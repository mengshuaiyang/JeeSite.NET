## 🎯 C# 编码规范

### 命名规范

| 类型 | 规则 | 示例 |
|------|------|------|
| 命名空间 | PascalCase | `JeeSiteNET.Core` |
| 类 / 接口 | PascalCase | `UserService`, `IUserRepository` |
| 方法 | PascalCase | `GetUserAsync` |
| 属性 | PascalCase | `UserName` |
| 字段 | _camelCase | `_userRepository` |
| 局部变量 | camelCase | `userName` |
| 常量 | UPPER_SNAKE_CASE | `MAX_PAGE_SIZE` |

### 文件范围命名空间

```csharp
namespace JeeSiteNET.Core;

public class ApiResult<T>
{
    // ...
}
```

### 异步方法命名约定

所有异步方法必须以 `Async` 后缀：

```csharp
public async Task<UserDto> GetUserAsync(string userCode)
{
    // ...
}
```

### 类型使用推荐

| 场景 | 推荐 | 避免 |
|------|------|------|
| 字符串空检查 | `string.IsNullOrEmpty()` | `== null` |
| 可空类型 | `T?` | `Nullable<T>` |
| 集合 | `List<T>`, `IQueryable<T>` | `ArrayList`, `IEnumerable<T>` (过度使用) |

## 🌐 API 设计规范

### 路由规则

```
api/v1/{module}/{controller}/{action?}
```

示例：
- `GET /api/v1/sys/user/page`
- `POST /api/v1/sys/user`
- `PUT /api/v1/sys/user/admin`
- `DELETE /api/v1/sys/user/admin`

### HTTP 方法使用

| 方法 | 用途 |
|------|------|
| GET | 查询数据（幂等） |
| POST | 新增数据 |
| PUT | 更新数据（幂等） |
| DELETE | 删除数据（幂等） |

### 统一响应

所有 API 返回统一格式 `ApiResult<T>`：

```json
{
  "Code": 200,
  "Message": "操作成功",
  "Data": { ... }
}
```

### 错误码规范

| 错误码 | 含义 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或登录过期 |
| 403 | 无操作权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 💾 数据库设计规范

### 表命名

```
{ModulePrefix}_{TableName}
```

示例：
- `Sys_User` / `Sys_Role` / `Sys_Menu`
- `Cms_Article` / `Cms_Category`
- `Bpm_ApprovalRecord`

### 字段命名

| 类型 | 规则 | 示例 |
|------|------|------|
| 主键 | `Id` 或 `{TableName}Id` | `UserId`, `ArticleId` |
| 外键 | `{TableName}Id` | `RoleId`, `ParentId` |
| 编码 | `{FieldName}Code` | `UserCode`, `DeptCode` |
| 名称 | `{FieldName}Name` | `UserName`, `DeptName` |
| 创建人/时间 | `CreateBy` / `CreateDate` | |
| 更新人/时间 | `UpdateBy` / `UpdateDate` | |
| 备注 | `Remarks` | |
| 状态 | `Status` | 0=正常, 1=禁用 |

### 数据类型映射

| .NET 类型 | SQL Server 类型 | 说明 |
|-----------|----------------|------|
| `string` | `NVARCHAR(n)` | 文本 |
| `int` | `INT` | 整数 |
| `long` | `BIGINT` | 长整数 |
| `decimal` | `DECIMAL(18,4)` | 小数 |
| `DateTime` | `DATETIME` | 日期时间 |
| `bool` | `BIT` | 布尔值 |
| `Guid` | `UNIQUEIDENTIFIER` | 唯一标识符 |

### 索引规范

| 场景 | 索引类型 |
|------|---------|
| 主键 | 聚集索引 (CLUSTERED) |
| 外键 | 非聚集索引 |
| 常用查询字段 | 非聚集索引 |
| 联合查询 | 复合索引 |

## 🎨 前端编码规范

### Vue 组件

- 组件名使用 PascalCase
- 文件命名使用 PascalCase
- 模板使用 2 空格缩进

```vue
<template>
  <div class="user-list">
    <!-- 内容 -->
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue'
const users = ref([])
</script>

<style scoped>
.user-list { padding: 16px; }
</style>
```

### TypeScript

- 接口命名使用 PascalCase + `Type` 后缀
- 类型定义使用 PascalCase

```typescript
interface UserType {
  userCode: string
  userName: string
  email?: string
}
```

### API 调用

统一使用 axios 封装：

```typescript
import request from './request'

export async function getUserList(params: PageRequest) {
  return request.get('/api/v1/sys/user/page', { params })
}
```

## ✅ 代码审查规范

### PR 提交要求

1. **描述清晰**: PR 标题和描述应清晰说明改动内容
2. **测试覆盖**: 新增功能应有对应的单元测试
3. **代码格式**: 符合项目编码规范
4. **冲突解决**: 确保无代码冲突
5. **文档更新**: 相关文档应同步更新

### 代码审查要点

| 检查项 | 说明 |
|--------|------|
| 命名规范 | 变量、函数、类命名是否符合规范 |
| 代码复用 | 是否存在重复代码 |
| 错误处理 | 是否有完善的异常处理 |
| 性能优化 | 是否有性能问题 |
| 安全漏洞 | 是否存在安全风险（SQL 注入/XSS/权限绕过等） |
| 测试覆盖 | 单元测试是否覆盖关键逻辑 |

## 🔬 单元测试规范

### 测试命名

```csharp
public class UserServiceTests
{
    [TestMethod]
    public async Task GetUserAsync_WithValidUserCode_ReturnsUser()
    {
        // Arrange
        // Act
        // Assert
    }
}
```

### 测试工具

| 用途 | 推荐库 |
|------|--------|
| 测试框架 | xUnit |
| Mock | NSubstitute / Moq |
| 断言 | FluentAssertions / 内置 |
| 内存数据库 | SQLite in-memory / EF Core in-memory |

### 测试金字塔

| 层级 | 占比 | 说明 |
|------|------|------|
| 单元测试 | ~70% | 测试单个组件/类/方法 |
| 集成测试 | ~20% | 测试组件之间的协作 |
| 端到端测试 | ~10% | 测试完整业务流程 |

## 📝 日志规范

### 日志级别

| 级别 | 用途 |
|------|------|
| Trace | 详细调试信息 |
| Debug | 调试信息 |
| Info | 业务流程信息 |
| Warn | 警告信息 |
| Error | 错误信息 |
| Critical | 严重错误 |

### 日志格式

```csharp
_logger.LogInformation("用户登录: {UserCode}", userCode);
_logger.LogError(ex, "用户查询失败: {UserCode}", userCode);
```

### 敏感信息

**禁止记录敏感信息：密码、密钥、个人隐私数据（身份证/手机号等）**

## 🔒 安全规范

### 输入验证

- 使用数据注解（DataAnnotations）验证
- 使用 FluentValidation 进行复杂验证
- 防止 SQL 注入（始终使用参数化查询）
- 防止 XSS（富文本需经 HtmlSanitizer 清洗）

```csharp
[Required]
[StringLength(50)]
public string UserCode { get; set; }
```

### 身份认证

- 使用 JWT Token
- Token 过期时间合理设置（默认 12 小时）
- 敏感接口需要权限验证
- 支持 Token 吊销（黑名单机制）

### 数据加密

- 密码使用 BCrypt 加密存储
- 敏感数据传输使用 HTTPS
- 数据库敏感字段可加密存储
- 国密算法（SM2/SM3/SM4）支持

## ⚡ 性能规范

### 数据库优化

- 使用索引优化查询
- 避免 `SELECT *`
- 必须使用分页查询
- 合理使用缓存

### 代码优化

- 避免嵌套循环
- 使用异步编程 (`async/await`)
- 合理使用 LINQ
- 避免在循环中进行数据库查询（N+1 查询问题）

### 缓存策略

- 使用 FusionCache (L1 内存 + L2 Redis)
- 设置合理的缓存过期时间
- 定义缓存失效策略（订阅/主动作废）
- 防止缓存穿透/雪崩/击穿

## 🚢 部署规范

### 环境配置

| 环境 | 用途 |
|------|------|
| Development | 开发环境 |
| Staging | 测试/预发环境 |
| Production | 生产环境 |

### 配置管理

- 使用环境变量覆盖默认配置
- 敏感配置使用密钥管理（dotnet user-secrets / Azure Key Vault 等）
- 配置文件不应包含敏感信息

### Docker 部署

```bash
# 构建镜像
docker build -t jeesite-webapi .

# 运行容器
docker run -p 5000:80 jeesite-webapi
```

## 📚 Git 规范

### 分支管理

| 分支 | 用途 |
|------|------|
| `main` / `master` | 主分支，稳定版本 |
| `develop` | 开发分支，整合功能 |
| `feature/*` | 功能分支 |
| `bugfix/*` | Bug 修复分支 |
| `hotfix/*` | 紧急修复分支 |

### Commit 消息规范

```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

**类型**:
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建/工具更新

**示例**:
```
feat(sys): 添加用户导出功能

- 新增用户列表导出接口
- 添加 Excel 导出服务
- 更新前端导出按钮
```
