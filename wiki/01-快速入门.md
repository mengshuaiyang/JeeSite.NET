# 01. 快速入门

> 本文档介绍如何快速搭建和运行 JeeSite.NET 项目。
> 最后更新: 2026-06-12

---

## 🎯 环境要求

### 后端开发环境

| 组件 | 版本要求 | 说明 |
|------|---------|------|
| .NET SDK | 10.0+ | 核心运行时 |
| Node.js | 18+ | 前端构建 |
| pnpm | 8+ | 推荐的包管理器 |

### 数据库

选择其一：
- **SQL Server**: 2022+（默认）
- **Sqlite**: 零配置开发环境
- **PostgreSQL**: 14+ / GaussDB
- **达梦 DM8** / **人大金仓 KingbaseES**

**Redis**: 7+（缓存，可选，降级为内存缓存）

---

## 🚀 快速启动

### 使用 Docker Compose（推荐）

```bash
# 进入项目根目录
cd d:\Projects\jeesite.net

# 启动全部服务（SQL Server + Redis + MinIO + Elasticsearch + WebAPI + 前端）
docker compose up -d
```

服务启动后：
- **Web API**: http://localhost:5000
- **前端**: http://localhost:3000
- **Swagger**: http://localhost:5000/swagger

### 本地开发

#### 1. 启动基础设施

```bash
# 启动 SQL Server 和 Redis
docker compose up -d sqlserver redis
```

#### 2. 运行后端 API

```bash
cd src/JeeSiteNET.Web.Api
dotnet restore
dotnet build
dotnet run
```

API 将在以下地址启动：
- HTTP: http://localhost:5000
- HTTPS: https://localhost:5001
- Swagger: http://localhost:5000/swagger

#### 3. 运行前端

```bash
cd frontend
pnpm install
pnpm dev
```

前端将在 http://localhost:5173 启动。

## 💾 数据库配置

在 `appsettings.json` 中配置：

```json
{
  "DatabaseProvider": "SqlServer",
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost;Database=JeeSiteNET;User ID=sa;Password=YourStrong!Passw0rd;TrustServerCertificate=True",
    "Redis": "localhost:6379"
  }
}
```

支持的 `DatabaseProvider` 值: `SqlServer`, `Sqlite`, `PostgreSQL`, `Dm`, `KingbaseES`

### 数据库迁移

```bash
# 默认 (SQL Server)
cd src/JeeSiteNET.Infrastructure
dotnet ef migrations add Init -- "Server=localhost;Database=JeeSiteNET;..."

# 指定 Provider
dotnet ef migrations add Init -- "Host=localhost;Database=JeeSiteNET;..." "PostgreSQL"
```

## 🛠️ 开发工具

### 代码生成器

系统内置代码生成器，支持多数据库表发现和代码下载：
- `POST /api/v1/codegen/table/generate` — 生成代码到磁盘
- `GET /api/v1/codegen/table/download` — 打包 ZIP 下载

详细使用说明请参考 [CodeGen代码生成模块](05-CodeGen代码生成模块)。

### Swagger 文档

API 文档访问地址：http://localhost:5000/swagger

### 系统监控

- 服务器状态: `GET /api/v1/sys/monitor/server`
- 在线用户: `POST /api/v1/sys/online/list`
- 缓存管理: `GET /api/v1/sys/cache/list`
- Elasticsearch: `http://localhost:9200/_cluster/health`

## ❓ 常见问题

### Q1. 忘记 admin 密码怎么办？

有三种方式：
1. 使用"忘记密码"页面（需配置邮件服务）
2. 使用管理员账号的短信/邮件验证码登录
3. 直接在数据库执行密码重置

### Q2. 新增的菜单/功能为什么用户看不到？

常见原因有三种:
- 菜单的"显示/隐藏"属性设置错误
- 用户的角色未包含该菜单权限
- 用户登录后未刷新缓存，退出重新登录即可

### Q3. 端口被占用怎么办？

```bash
# 查找占用端口的进程
netstat -ano | findstr :5000

# 终止进程
taskkill /F /PID <进程ID>
```

### Q4. 数据库连接失败？

请确保数据库服务已启动，并检查 `appsettings.json` 中的 `DatabaseProvider` 和连接字符串配置正确。

### Q5. 如何切换国产数据库？

修改 `appsettings.json` 中的 `DatabaseProvider`，安装对应的 NuGet 包。详细说明请参考 [系统管理员操作手册](29-系统管理员操作手册)。
