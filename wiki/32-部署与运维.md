## 🚀 部署方式

### 方式一：Docker Compose（推荐）

#### 启动服务

```bash
# 进入项目目录
cd d:\Projects\jeesite.net

# 启动所有服务
docker compose up -d
```

#### 服务列表

| 服务 | 端口 | 说明 |
|------|------|------|
| SQL Server | 1433 | 主数据库 |
| Redis | 6379 | 缓存 |
| MinIO | 9000 | 对象存储（文件上传） |
| Elasticsearch | 9200 | 全文搜索 |
| Web API | 5000 | 后端 API 服务 |
| 前端（Nginx） | 3000 | 前端 Web 应用 |
| Loki | 3100 | 日志聚合（可选） |
| Grafana | 3001 | 监控与可视化（可选） |

#### docker-compose.yml 配置示例

```yaml
services:
  sqlserver:
    image: mcr.microsoft.com/mssql/server:2022-latest
    environment:
      ACCEPT_EULA: Y
      SA_PASSWORD: ${SA_PASSWORD:-YourStrong!Passw0rd}
    ports:
      - "1433:1433"
    volumes:
      - sqlserver_data:/var/opt/mssql

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data

  webapi:
    build:
      context: .
      dockerfile: src/JeeSiteNET.Web.Api/Dockerfile
    ports:
      - "5000:80"
    environment:
      - ConnectionStrings__DefaultConnection=Server=sqlserver;Database=JeeSiteNET;User ID=sa;Password=YourStrong!Passw0rd;TrustServerCertificate=True
      - ConnectionStrings__Redis=redis:6379
      - ASPNETCORE_ENVIRONMENT=Production
    depends_on:
      - sqlserver
      - redis

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    ports:
      - "3000:80"
    depends_on:
      - webapi

volumes:
  sqlserver_data:
  redis_data:
```

### 方式二：手动部署（IIS / Kestrel / Nginx）

#### 1. 部署后端 API

```bash
# 进入项目目录
cd d:\Projects\jeesite.net

# 构建项目
dotnet build -c Release

# 发布项目
dotnet publish src/JeeSiteNET.Web.Api -c Release -o publish

# 运行项目 (Kestrel)
cd publish
dotnet JeeSiteNET.Web.Api.dll
```

#### 2. 部署前端

```bash
cd frontend

# 安装依赖
pnpm install

# 构建生产版本
pnpm build

# 部署到 Nginx
cp -r dist/* /usr/share/nginx/html/
```

#### 3. 配置 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://localhost:5000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # SignalR 实时通信
    location /hub/ {
        proxy_pass http://localhost:5000/hub/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```

#### 4. HTTPS 配置（推荐）

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    # 其他配置同上
}
```

## 🔒 安全配置

### 修改默认密码

**生产环境部署后，请立即执行以下操作：**

1. 修改 `admin` 账号密码（登录系统后进入用户管理）
2. 修改 SQL Server `sa` 账号密码
3. 修改 Redis 访问密码
4. 修改 JWT Secret Key
5. 修改 MinIO 访问密钥
6. 更新所有数据库/缓存连接字符串

### HTTPS 强制

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$host$request_uri;
}
```

### 防火墙规则

只开放必要端口：
- **80** / **443** (前端/API)
- **1433** (SQL Server，应限制内网访问)
- **6379** (Redis，应限制内网访问)
- **9200** (Elasticsearch，应限制内网访问)
- **9000** (MinIO，应限制内网访问)

## 🔄 备份与恢复

### 数据库备份

```bash
# SQL Server 完整备份
sqlcmd -S localhost -U sa -P YourStrong!Passw0rd \
  -Q "BACKUP DATABASE [JeeSiteNET] TO DISK = 'D:\Backup\JeeSiteNET.bak' WITH INIT"

# 差异备份
sqlcmd -S localhost -U sa -P YourStrong!Passw0rd \
  -Q "BACKUP DATABASE [JeeSiteNET] TO DISK = 'D:\Backup\JeeSiteNET_diff.bak' WITH DIFFERENTIAL"

# 数据库恢复
sqlcmd -S localhost -U sa -P YourStrong!Passw0rd \
  -Q "RESTORE DATABASE [JeeSiteNET] FROM DISK = 'D:\Backup\JeeSiteNET.bak'"
```

### Redis 数据持久化

Redis 默认开启 RDB 快照，容器重启后自动恢复。建议定期备份 Redis 数据目录。

### 文件备份

```bash
# 备份上传文件
tar -czf /backup/files_$(date +%Y%m%d).tar.gz /var/www/files

# 备份配置文件
cp appsettings.json /backup/appsettings_$(date +%Y%m%d).json
```

### 备份策略建议

| 备份类型 | 频率 | 保留时长 |
|---------|------|---------|
| 数据库完整备份 | 每日 02:00 | 30 天 |
| 数据库差异备份 | 每小时 | 7 天 |
| Redis 快照 | 自动（RDB） | 随容器 |
| 配置文件 | 每次更新 | 永久 |
| 上传文件 | 每日 | 永久 |

## ⚡ 性能优化

### 配置建议

| 配置项 | 建议值 | 说明 |
|--------|--------|------|
| 数据库连接池大小 | 50-100 | max pool size |
| Redis 最大连接数 | 100 | 缓存并发连接 |
| 请求超时时间 | 30 秒 | HTTP 请求超时 |
| 响应压缩 | 启用 | Gzip / Brotli |
| 静态文件缓存 | 启用 | Nginx / CDN |
| L1 缓存过期 | 5 分钟 | 内存缓存 |
| L2 缓存过期 | 30 分钟 | Redis 缓存 |

### Nginx 负载均衡

```nginx
http {
    upstream api_backend {
        server localhost:5000;
        server localhost:5001;
        server localhost:5002;
    }

    server {
        location /api/ {
            proxy_pass http://api_backend;
        }
    }
```

## 🔄 升级指南

### 升级步骤

1. **备份数据库**：在升级前执行完整备份
2. **备份配置文件**：备份 `appsettings.json` 和前端 `.env`
3. **拉取最新代码**：`git pull origin master`
4. **执行数据库迁移**：`dotnet ef database update`
5. **构建并部署新代码**：`dotnet publish` 或 `docker compose build --no-cache`
6. **验证服务**：访问健康检查端点与首页，检查日志

### 回滚操作

如升级后出现问题：

```bash
# 1. 停止新版本
docker compose stop webapi frontend

# 2. 恢复数据库
sqlcmd -S localhost -U sa -P YourStrong!Passw0rd \
  -Q "RESTORE DATABASE [JeeSiteNET] FROM DISK = 'D:\Backup\JeeSiteNET.bak'"

# 3. 回滚代码
git checkout <previous-commit>

# 4. 重新构建部署
docker compose build webapi frontend
docker compose up -d
```

## 🔍 故障排查

### 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 数据库连接失败 | 连接字符串错误 / 服务未启动 | 检查 appsettings.json / 启动 SQL Server |
| API 无法启动 | 端口被占用 / 依赖缺失 | 检查 5000 端口 / 执行 dotnet restore |
| 前端无法访问 API | CORS 配置错误 / 反向代理失效 | 检查 Nginx / Startup 配置 |
| Redis 连接失败 | Redis 服务未启动 / 密码错误 | 启动 Redis / 检查连接字符串 |
| 登录后无菜单 | 菜单权限未分配 / 缓存未刷新 | 以 admin 登录分配角色权限 / 清理缓存 |
| 文件上传失败 | 文件大小超限 / 存储服务未配置 | 检查 upload.maxSize / MinIO 配置 |
| SignalR 连接失败 | WebSocket 代理未配置 | 检查 Nginx Upgrade/Connection 头 |

### 日志收集与分析

故障时请准备以下信息：
- `logs/*.log` 中的错误日志
- 浏览器 DevTools → Console 输出
- 可复现步骤
- 环境信息（操作系统 / .NET 版本 / 数据库版本）

## ❓ 常见问题（3-5 个）

1. **问：如何实现零停机部署？**  
   答：使用 `docker compose up -d --no-deps` 逐个更新服务，或使用 Swarm/Kubernetes 的 rolling update 策略。

2. **问：数据库如何备份和恢复？**  
   答：每日定时备份到对象存储（MinIO/S3），恢复时先停止 WebAPI，恢复数据库，再启动服务。

3. **问：如何监控生产环境的健康状态？**  
   答：Grafana 仪表盘 + Loki 日志 + Prometheus 指标采集，设置 CPU/内存/磁盘/RPS/错误率等告警阈值。
