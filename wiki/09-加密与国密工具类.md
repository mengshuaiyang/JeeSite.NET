# 工具类手册 · 加密与国密工具

## 一、工具类定位

JeeSite.NET 在 `src/JeeSiteNET.Core/Utils/` 目录下提供了一组加密与国密工具类，覆盖对称加密、非对称加密、哈希、Base64 编解码以及国家商用密码算法（SM2/SM4）。

| 工具类 | 文件路径 | 职责 | 依赖 |
|--------|---------|------|------|
| `EncryptUtil` | `Utils/EncryptUtil.cs` | 通用加密工具（AES / DES / RSA / MD5 / SHA1 / SHA256 / SHA512 / Base64） | System.Security.Cryptography |
| `Sm2Util` | `Utils/Sm2Util.cs` | SM2 国密椭圆曲线算法（签名 + 非对称加解密） | BouncyCastle.Cryptography 2.6+ |
| `Sm4Util` | `Utils/Sm4Util.cs` | SM4 国密对称加密（CBC 模式 + PKCS7 填充） | BouncyCastle.Cryptography 2.6+ |
| `RsaUtil` | `Utils/RsaUtil.cs` | 独立的 RSA 工具（PEM 导入导出 / XML 互转） | System.Security.Cryptography |

> **术语速查**
> - **PKCS7 填充**：块密码在最后一个数据块不足时的标准填充方式，每个填充字节值 = 填充字节数。
> - **CBC 模式**：密文块链模式，需要 IV（初始向量），相同明文 + 不同 IV → 不同密文。
> - **C1C2C3 拼接**：SM2 加密结果的标准格式，C1=密文点（65 字节），C2=密文（与明文等长），C3=SM3 哈希（32 字节）。

## 二、EncryptUtil API 参考

`EncryptUtil` 是项目中最基础的加密工具，封装了 .NET 原生的 `System.Security.Cryptography` API。所有方法均为 `public static`。

### 2.1 AES 对称加密

**签名**
```csharp
public static string EncryptAes(string plainText, byte[] key, byte[] iv)
public static string DecryptAes(string cipherText, byte[] key, byte[] iv)
```

**作用**
- `EncryptAes`: 使用 AES-CBC-PKCS7 加密 UTF-8 明文，返回 Base64 编码的密文字符串。
- `DecryptAes`: 将 Base64 密文字符串解密回 UTF-8 明文。
- `key` 长度必须为 16 / 24 / 32 字节（对应 AES-128 / AES-192 / AES-256）。
- `iv` 必须为 16 字节。

**示例**
```csharp
var key = EncryptUtil.GenerateRandomKey(128);   // 16 字节
var iv  = EncryptUtil.GenerateRandomKey(128);   // 16 字节
var encrypted = EncryptUtil.EncryptAes("hello world", key, iv);
var decrypted = EncryptUtil.DecryptAes(encrypted, key, iv);
// decrypted == "hello world"
```

### 2.2 RSA 非对称加密

**签名**
```csharp
public static string EncryptRsa(string plainText, string publicKey)
public static string DecryptRsa(string cipherText, string privateKey)
```

**作用**
- `EncryptRsa`: 使用 RSA 公钥加密明文，返回 Base64 密文。支持 PEM / XML 两种公钥格式。
- `DecryptRsa`: 使用 RSA 私钥解密 Base64 密文。
- 加密分块：RSA 单次加密明文长度受限（密钥长度 / 8 - 填充开销），长文本内部自动分块加密并拼接。

**示例**
```csharp
var keys = RsaUtil.GeneratePemKeyPair(2048);
var enc = EncryptUtil.EncryptRsa("敏感数据", keys.PublicKey);
var dec = EncryptUtil.DecryptRsa(enc, keys.PrivateKey);
```

### 2.3 哈希算法

**签名**
```csharp
public static string Md5(string input)
public static string Sha1(string input)
public static string Sha256(string input)
public static string Sha512(string input)
```

**作用**
- 分别计算 MD5 / SHA1 / SHA256 / SHA512 哈希值，输入 UTF-8 字符串，输出**32 / 40 / 64 / 128 位小写十六进制**字符串。
- `Md5` 仅用于文件完整性校验，**严禁用于密码存储**。
- 密码存储请使用 `PasswordHasher`（ASP.NET Core Identity 提供的 PBKDF2 实现）。

**示例**
```csharp
var md5    = EncryptUtil.Md5("abc");       // "900150983cd24fb0d6963f7d28e17f72"
var sha1   = EncryptUtil.Sha1("abc");      // "a9993e364706816aba3e25717850c26c9cd0d89d"
var sha256 = EncryptUtil.Sha256("abc");    // "ba7816bf8f01cfea414140de5dae2223b00361a396177a9cb410ff61f20015ad"
```

### 2.4 Base64 编解码

**签名**
```csharp
public static string EncodeBase64(byte[] bytes)
public static byte[] DecodeBase64(string base64Str)
```

**作用**
- `EncodeBase64`: 将字节数组编码为 Base64 字符串。
- `DecodeBase64`: 将 Base64 字符串还原为字节数组，非法输入抛出 `FormatException`。

**示例**
```csharp
var b64 = EncryptUtil.EncodeBase64(Encoding.UTF8.GetBytes("hello"));  // "aGVsbG8="
var raw = EncryptUtil.DecodeBase64(b64);                               // byte[5]
```

### 2.5 随机密钥生成

**签名**
```csharp
public static byte[] GenerateRandomKey(int bits)
```

**作用**
- 使用加密安全随机数生成器（`RandomNumberGenerator`）生成 `bits / 8` 字节的随机密钥。
- 常用取值：`128`（AES-128 / SM4）、`192`（AES-192）、`256`（AES-256 / SM2 私钥）。

**示例**
```csharp
var aesKey = EncryptUtil.GenerateRandomKey(128);  // byte[16]
var iv     = EncryptUtil.GenerateRandomKey(128);  // byte[16]
```

## 三、Sm2Util API 参考

`Sm2Util` 基于 BouncyCastle Cryptography 2.6 实现 SM2 椭圆曲线算法（国密局推荐，曲线参数 `sm2p256v1`）。所有方法均为 `public static`。

### 3.1 密钥对生成

**签名**
```csharp
public static (byte[] publicKey, byte[] privateKey) GenerateKeyPair()
```

**作用**
- 生成一组 SM2 密钥对。
- **公钥**：65 字节 `0x04 + X(32B) + Y(32B)`（未压缩格式）。
- **私钥**：32 字节大整数。
- 返回值建议立即转换为 Base64 字符串持久化（见下文配置示例）。

**示例**
```csharp
var (pub, priv) = Sm2Util.GenerateKeyPair();
string pubB64  = Convert.ToBase64String(pub);   // 65 字节 → 88 字符 Base64
string privB64 = Convert.ToBase64String(priv);  // 32 字节 → 44 字符 Base64
```

### 3.2 SM3withSM2 签名

**签名**
```csharp
public static string Sign(byte[] privateKey, string plainText)
public static bool Verify(byte[] publicKey, string plainText, string signature)
```

**作用**
- `Sign`: 使用 SM2 私钥对 UTF-8 明文做 SM3withSM2 签名，返回 Base64 字符串。
- `Verify`: 使用 SM2 公钥验证签名是否匹配指定明文。

**示例**
```csharp
var (pub, priv) = Sm2Util.GenerateKeyPair();
var sig = Sm2Util.Sign(priv, "待签名字符串");
bool ok = Sm2Util.Verify(pub, "待签名字符串", sig);  // true
```

### 3.3 SM2 非对称加解密

**签名**
```csharp
public static string Encrypt(byte[] publicKey, string plainText)
public static string Decrypt(byte[] privateKey, string cipherText)
```

**作用**
- `Encrypt`: 使用 SM2 公钥加密 UTF-8 明文，密文格式为 **C1C2C3**（BouncyCastle 默认 `C1C3C2` 由内部调整），返回 Base64。
- `Decrypt`: 使用 SM2 私钥解密 Base64 密文，返回 UTF-8 明文。

**示例**
```csharp
var (pub, priv) = Sm2Util.GenerateKeyPair();
var enc = Sm2Util.Encrypt(pub, "身份证号/手机号等敏感字段");
var dec = Sm2Util.Decrypt(priv, enc);
```

### 3.4 注意事项

- 公钥 65 字节必须以 `0x04` 开头（未压缩点），否则视为非法。
- 私钥 32 字节必须是 `[1, n-1]` 之间的大整数（BouncyCastle 会校验）。
- 跨语言互操作时注意：Java 使用 `C1C3C2`，.NET BouncyCastle 默认也是 `C1C3C2`。若与使用 `C1C2C3` 约定的系统对接，需在 `Sm2Util` 内部调整字节顺序。

## 四、Sm4Util API 参考

`Sm4Util` 基于 BouncyCastle 实现 SM4 对称加密（国密替代 AES），默认 **CBC 模式 + PKCS7 填充**。

### 4.1 密钥生成

**签名**
```csharp
public static string GenerateKey()
```

**作用**
- 生成 128 位（16 字节）随机 SM4 密钥，返回 Base64 字符串（约 24 字符）。

**示例**
```csharp
string sm4Key = Sm4Util.GenerateKey();  // "xK9mP2vQ8rA1..."
```

### 4.2 CBC 模式加解密

**签名**
```csharp
public static string Encrypt(string keyBase64, string plainText)
public static string Encrypt(string keyBase64, string plainText, byte[] iv)
public static string Decrypt(string keyBase64, string cipherText)
public static string Decrypt(string keyBase64, string cipherText, byte[] iv)
```

**作用**
- `Encrypt`: 将 UTF-8 明文加密为 Base64 密文。
- `Decrypt`: 将 Base64 密文解密为 UTF-8 明文。
- 不带 `iv` 参数的重载使用默认 IV = 16 字节 `0x00`。
- 显式传入 `iv`（16 字节）可获得更高安全性（相同明文会产出不同密文）。

**示例**
```csharp
string key = Sm4Util.GenerateKey();
string enc = Sm4Util.Encrypt(key, "敏感数据");
string dec = Sm4Util.Decrypt(key, enc);

// 推荐：显式指定 IV
byte[] iv = EncryptUtil.GenerateRandomKey(128);
string enc2 = Sm4Util.Encrypt(key, "敏感数据", iv);
string dec2 = Sm4Util.Decrypt(key, enc2, iv);
```

## 五、在 appsettings.json 中配置默认密钥

建议在 `appsettings.json` 中集中配置加密密钥，然后通过 `IOptions<CryptographyOptions>` 注入使用。生产环境下密钥必须从配置文件中**抽离**（见第七节）。

```json
{
  "Cryptography": {
    "AesKey": "Base64Encoded16Bytes",
    "AesIv": "Base64Encoded16Bytes",
    "RsaPrivateKey": "-----BEGIN RSA PRIVATE KEY-----\nMIIE...\n-----END RSA PRIVATE KEY-----",
    "RsaPublicKey": "-----BEGIN PUBLIC KEY-----\nMIIB...\n-----END PUBLIC KEY-----",
    "Sm2PrivateKey": "Base64Encoded32Bytes",
    "Sm2PublicKey": "Base64Encoded65Bytes",
    "Sm4Key": "Base64Encoded16Bytes"
  }
}
```

对应的 `CryptographyOptions` 配置类（建议放在 `Core/Options/` 目录下）：
```csharp
public class CryptographyOptions
{
    public string AesKey { get; set; }
    public string AesIv { get; set; }
    public string RsaPrivateKey { get; set; }
    public string RsaPublicKey { get; set; }
    public string Sm2PrivateKey { get; set; }
    public string Sm2PublicKey { get; set; }
    public string Sm4Key { get; set; }
}
```

注册方式：
```csharp
services.Configure<CryptographyOptions>(configuration.GetSection("Cryptography"));
```

使用方式：
```csharp
public class UserService
{
    private readonly CryptographyOptions _crypto;

    public UserService(IOptions<CryptographyOptions> crypto)
    {
        _crypto = crypto.Value;
    }

    public string EncryptMobile(string mobile)
    {
        // 使用 SM4 加密手机号
        return Sm4Util.Encrypt(_crypto.Sm4Key, mobile);
    }
}
```

## 六、国密使用场景

国密算法（SM2 / SM3 / SM4）由国家密码管理局发布，在涉及敏感数据的政务、金融、医疗等场景中往往是**合规硬性要求**。

| 算法 | 替代对象 | 典型使用场景 |
|------|---------|------------|
| **SM2**（非对称） | RSA-2048 / ECDSA | 数字签名（接口响应签名 / 登录票据签名）、非对称加密（密钥交换 / 小数据加密） |
| **SM3**（哈希） | SHA-256 | 文件完整性校验、消息摘要、作为 SM2 签名的底层哈希 |
| **SM4**（对称） | AES-128 | 大批量敏感字段加解密（身份证号、手机号、邮箱、银行卡号等） |

**敏感信息数据库加密存储（推荐方案）**
```csharp
// DTO 进入仓储前加密
entity.IdCard   = Sm4Util.Encrypt(_crypto.Sm4Key, dto.IdCard);
entity.Mobile   = Sm4Util.Encrypt(_crypto.Sm4Key, dto.Mobile);
entity.Email    = Sm4Util.Encrypt(_crypto.Sm4Key, dto.Email);

// 读出后解密（ValueConverter 亦可自动完成）
dto.IdCard = Sm4Util.Decrypt(_crypto.Sm4Key, entity.IdCard);
```

> 建议使用 EF Core 的 `ValueConverter` 在 DbContext 层面自动完成加解密，业务代码无感知。

## 七、密钥管理最佳实践

### 7.1 密钥与代码分离

- **严禁**将真实生产密钥提交到 Git 仓库。
- 生产环境使用以下任意一种密钥管理方案：
  - **Azure Key Vault**（Azure 托管）
  - **HashiCorp Vault**（自建开源方案）
  - **Kubernetes Secrets**（K8s 环境）
  - **环境变量**（最小化方案，避免写入磁盘）
- 本地开发：使用 `dotnet user-secrets` 或 `appsettings.Development.json`（已在 `.gitignore` 中排除）。

### 7.2 密钥轮换策略

- 建议每 **90 天** 轮换一次密钥。
- 轮换流程：
  1. 生成新密钥。
  2. 配置文件 / 密钥库同时保留旧密钥（解密用）+ 新密钥（加密用）。
  3. 后台作业遍历已加密数据，用旧密钥解密 → 用新密钥加密 → 写回。
  4. 全量数据迁移完成后移除旧密钥。
- 建议在数据库中保留加密版本号字段（如 `crypto_version`），便于追踪每条记录使用的密钥版本。

### 7.3 多环境隔离

| 环境 | 密钥策略 |
|------|---------|
| **开发（Development）** | 使用固定示例密钥（仅开发机器可见） |
| **测试（Staging）** | 使用独立测试密钥，与生产密钥无复用 |
| **生产（Production）** | 使用密钥管理服务托管，开发人员不可见 |

### 7.4 审计与监控

- 对密钥的访问行为进行审计日志记录（谁、何时、何地访问了哪个密钥）。
- 定期自动检测密钥即将过期并发出预警。
- 敏感字段加解密失败必须单独记录日志（注意：日志中不可写入密钥本身）。
