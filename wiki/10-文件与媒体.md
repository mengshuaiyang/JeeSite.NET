### 一、VideoUtil

`VideoUtil`（`Utils/VideoUtil.cs`）基于 **FFmpeg** 实现视频信息读取、缩略图生成、格式转码、码率压缩、音频提取等常用操作。

#### 1.1 FFmpeg 安装与配置

- Windows 下从 <https://www.gyan.dev/ffmpeg/builds/> 下载 `ffmpeg-release-essentials.zip`。
- 解压后将 `bin/` 目录加入 PATH，或在 `appsettings.json` 中显式指定：

```json
{
  "FFmpeg": {
    "Path": "C:\\tools\\ffmpeg\\bin\\ffmpeg.exe"
  }
}
```

- Linux / Docker 环境：`apt-get install -y ffmpeg`。

#### 1.2 核心方法

**（1）获取视频信息**
```csharp
public static async Task<VideoInfo> GetInfoAsync(string inputPath)
```
返回 `VideoInfo` 对象，包含：
- `Duration`：时长（`TimeSpan`）
- `Width` / `Height`：分辨率
- `VideoCodec` / `AudioCodec`：编码格式
- `FileSizeBytes`：文件大小
- `BitRateKbps`：码率（kbps）

**（2）生成缩略图**
```csharp
public static async Task GenerateThumbnailAsync(string inputPath, string outputPath, int seconds = 1)
```
在视频的第 `seconds` 秒截取一张 PNG 图像作为缩略图。

**（3）转码为 MP4**
```csharp
public static async Task ConvertToMp4Async(string inputPath, string outputPath)
```
将任意格式视频转码为 `H.264 + AAC` 的 MP4 容器，适合浏览器播放。

**（4）压缩到目标码率**
```csharp
public static async Task CompressAsync(string inputPath, string outputPath, int targetKbps = 1500)
```
将视频重新编码到指定码率，适合移动端播放或节省存储空间。

**（5）提取音频**
```csharp
public static async Task ExtractAudioAsync(string inputPath, string outputPath)
```
从视频中提取音轨为 MP3 文件。

#### 1.3 调用示例

```csharp
var info = await VideoUtil.GetInfoAsync("d:\\media\\input.mp4");
// info.Duration → 00:03:42
// info.Width    → 1920

await VideoUtil.GenerateThumbnailAsync(
    "d:\\media\\input.mp4",
    "d:\\media\\thumb.png",
    seconds: 5);

await VideoUtil.ConvertToMp4Async(
    "d:\\media\\input.mov",
    "d:\\media\\output.mp4");

await VideoUtil.CompressAsync(
    "d:\\media\\input.mp4",
    "d:\\media\\compressed.mp4",
    targetKbps: 800);

await VideoUtil.ExtractAudioAsync(
    "d:\\media\\input.mp4",
    "d:\\media\\audio.mp3");
```

#### 1.4 设计要点

- 所有方法均使用 `Process.Start` 调用 `ffmpeg.exe`，并通过 `-progress pipe:1` 读取进度（可扩展）。
- 生产环境建议通过 `IHostedService` + 队列异步处理，避免阻塞 HTTP 请求线程。
- 大文件转码可设置 `ProcessPriorityClass.Idle`，避免抢占前台资源。

### 二、ImageGeoUtil

`ImageGeoUtil`（`Utils/ImageGeoUtil.cs`）基于 **SixLabors.ImageSharp** 解析 JPEG 图片的 EXIF GPS 信息。

#### 2.1 依赖

```xml
<PackageReference Include="SixLabors.ImageSharp" Version="3.1.*" />
```

#### 2.2 核心方法

**（1）读取 GPS 信息**
```csharp
public static GeoInfo? ReadGeoInfo(string imagePath)
public static GeoInfo? ReadGeoInfo(Stream imageStream)
```

返回 `GeoInfo` 对象：
- `Latitude`：纬度（十进制度，正数表示北纬）
- `Longitude`：经度（十进制度，正数表示东经）
- `Altitude`：海拔（米，可为 `null`）
- `GpsTime`：GPS 时间戳（可为 `null`）

无 EXIF GPS 信息时返回 `null`。

**（2）度分秒 → 十进制度**
```csharp
public static double ConvertDmsToDecimal(double degrees, double minutes, double seconds, char reference)
```
- `reference` 取值：`'N'` / `'S'` / `'E'` / `'W'`
- 南半球 / 西半球 返回负数

#### 2.3 解析原理

EXIF 中 GPS 存储为 3 个 `RATIONAL`（分子/分母）对，分别表示度、分、秒：
- Tag `0x0002` = `GPSLatitude`
- Tag `0x0004` = `GPSLongitude`
- Tag `0x0006` = `GPSAltitude`
- Tag `0x0001` = `GPSLatitudeRef`（`"N"` / `"S"`）
- Tag `0x0003` = `GPSLongitudeRef`（`"E"` / `"W"`）

转换公式：`decimal = degrees + minutes/60 + seconds/3600`，再根据 Ref 决定正负。

#### 2.4 示例

```csharp
var geo = ImageGeoUtil.ReadGeoInfo("d:\\photos\\beijing.jpg");
if (geo != null)
{
    // { Latitude: 39.9042, Longitude: 116.4074, Altitude: 52.3 }
    Console.WriteLine(
        $"https://maps.google.com/?q={geo.Latitude},{geo.Longitude}");
}

// 手动转换
var lat = ImageGeoUtil.ConvertDmsToDecimal(39, 54, 15.1, 'N');  // ≈ 39.9042
var lng = ImageGeoUtil.ConvertDmsToDecimal(116, 24, 26.6, 'E'); // ≈ 116.4074
```

#### 2.5 隐私提示

上传到公开站点的图片建议调用 `ImageGeoUtil` 检测并**擦除 GPS 信息**，防止位置泄露。

### 三、FileSecurityUtil

`FileSecurityUtil`（`Utils/FileSecurityUtil.cs`）负责文件上传安全校验，采用**多层防御**策略：扩展名白名单 → 文件签名（Magic Number）校验 → 路径遍历检测 → 文件名清洗 → 大小限制。

#### 3.1 核心方法

**（1）扩展名白名单校验**
```csharp
public static bool IsExtensionSafe(string fileName, IEnumerable<string> allowedExtensions)
```
- 以 `.` 开头的扩展列表，不区分大小写。
- 注意：`file.exe.png` 将被认为扩展名是 `.png`，所以还要配合签名校验。

**（2）文件签名（Magic Number）校验**
```csharp
public static bool ValidateSignature(string fileName, Stream stream)
```
读取文件头部字节并与已知格式签名比对，防止伪造扩展名。
- JPEG：`FF D8 FF`
- PNG：`89 50 4E 47 0D 0A 1A 0A`
- GIF：`47 49 46 38`
- PDF：`25 50 44 46`
- ZIP / DOCX / XLSX / PPTX：`50 4B 03 04`
- MP4：偏移 4 字节处为 `66 74 79 70`（`"ftyp"`）

**（3）路径遍历检测**
```csharp
public static bool IsPathTraversalAttempt(string fileName)
```
识别包含 `../`、`..\\`、`/etc/passwd`、绝对路径（`C:\\`、`/`）等恶意模式。

**（4）文件名清洗**
```csharp
public static string SanitizeFileName(string fileName)
```
- 移除路径分隔符 `\` `/`
- 移除控制字符（ASCII < 0x20）
- 移除 Unicode 控制字符
- 替换空格为 `_`
- 截断超长文件名（默认 200 字符）

**（5）组合校验**
```csharp
public static (bool Success, string? Error) ValidateUpload(
    string fileName,
    Stream stream,
    long maxSizeBytes,
    IEnumerable<string>? allowedExtensions = null)
```
一次性执行大小限制 + 扩展名 + 签名 + 路径安全 + 安全文件名检查，返回 `(Success, Error)` 元组。

#### 3.2 默认白名单扩展名

```text
图片 : .jpg  .jpeg  .png  .gif  .bmp  .webp
文档 : .pdf  .doc  .docx  .xls  .xlsx  .ppt  .pptx  .txt  .md
压缩 : .zip  .7z  .rar
视频 : .mp4  .mov  .avi
```

#### 3.3 黑名单（严禁上传）

```text
.exe  .dll  .bat  .cmd  .ps1  .js  .sh  .php  .asp  .jsp  .htm  .html  .swf  .iso  .vbs  .wsf  .msi  .jar
```

#### 3.4 在 FileService / EditorController 中的典型用法

```csharp
var file = Request.Form.Files[0];
var check = FileSecurityUtil.ValidateUpload(
    file.FileName,
    file.OpenReadStream(),
    maxSizeBytes: 10 * 1024 * 1024);    // 10 MB

if (!check.Success)
{
    return BadRequest(new { error = check.Error });
}

var safeName = FileSecurityUtil.SanitizeFileName(file.FileName);
var finalPath = Path.Combine(_uploadDir, safeName);
// 进一步使用 GUID 前缀避免同名冲突
```

### 四、ChunkUploadService

`ChunkUploadService`（`Modules/Sys/Application/Services/ChunkUploadService.cs`）为大文件上传提供**分片 + 断点续传**能力。

#### 4.1 核心方法

**（1）初始化上传会话**
```csharp
public async Task<UploadSession> InitUpload(
    string sessionId,
    string fileName,
    long totalSize,
    int chunkSize = 5 * 1024 * 1024)
```
- `sessionId`：前端生成的唯一标识（建议使用 `文件 SHA256`）
- 在 `sys_file_upload` 表中记录会话信息：文件名、总大小、分片数量、已上传分片

**（2）上传单个分片**
```csharp
public async Task<ChunkResult> UploadChunk(
    string sessionId,
    int chunkIndex,
    Stream chunkStream)
```
- 将分片文件保存到临时目录 `{tempDir}/{sessionId}/{chunkIndex}`
- 原子更新 `sys_file_upload` 的 `uploaded_chunks` 字段

**（3）合并分片**
```csharp
public async Task<string> CompleteUpload(string sessionId)
```
- 校验所有分片是否完整
- 按顺序合并为最终文件
- 删除临时分片目录
- 返回最终文件的存储路径 / FileId

**（4）取消上传**
```csharp
public async Task CancelUpload(string sessionId)
```
删除临时文件 + 将会话标记为取消。

**（5）查询上传状态**
```csharp
public async Task<UploadStatus> GetUploadStatus(string sessionId)
```
返回已上传分片列表、总进度百分比、是否已完成，供前端断点续传时跳过已上传分片。

#### 4.2 推荐的前端流程

```text
1. 前端计算文件 SHA256 → 作为 sessionId
2. GET  /api/sys/chunk/status/{sessionId}
   ├─ 已存在 Complete = true  → 秒传，直接返回 FileId
   └─ 已存在 Complete = false → 跳过已上传分片
3. POST /api/sys/chunk/init   → 初始化会话（如不存在）
4. 并发上传分片（推荐 3 并发）
   POST /api/sys/chunk/upload  (multipart/form-data: sessionId + chunkIndex + file)
5. 全部完成后 POST /api/sys/chunk/complete → 服务端合并
```

#### 4.3 配置项

```json
{
  "FileUpload": {
    "TempDir": "D:\\uploads\\temp",
    "FinalDir": "D:\\uploads\\files",
    "DefaultChunkSizeMB": 5,
    "MaxFileSizeMB": 2048
  }
}
```

### 五、PreviewService

`PreviewService`（`Modules/Sys/Application/Services/PreviewService.cs`）提供 Office → PDF → HTML 的文档预览能力，依赖 **LibreOffice** 与 **Pdfium**。

#### 5.1 依赖安装

- **Windows**：下载 LibreOffice 便携版，指定 `soffice.exe` 路径
- **Linux / Docker**：`apt-get install -y libreoffice`
- Pdfium：使用 `PDFiumCore` NuGet 包（无需安装外部依赖）

#### 5.2 配置

```json
{
  "Preview": {
    "LibreOfficePath": "C:\\Program Files\\LibreOffice\\program\\soffice.exe",
    "TempDir": "D:\\uploads\\preview"
  }
}
```

#### 5.3 核心方法

**（1）Office 文档 → PDF**
```csharp
public async Task<string> ConvertToPdfAsync(string inputPath, string outputPath)
```
- 内部调用 `soffice --headless --convert-to pdf --outdir {outDir} {input}`
- 支持 `.doc` / `.docx` / `.xls` / `.xlsx` / `.ppt` / `.pptx` / `.txt` 等
- 成功返回 `outputPath`，失败抛出 `PreviewConvertException`

**（2）PDF / Office → HTML 预览**
```csharp
public async Task<string> ConvertToHtmlAsync(string inputPath, string outputPath)
```
- 若输入是 Office 文档，先经 LibreOffice 转 PDF
- 再用 Pdfium 将 PDF 逐页渲染为图片 + 生成 HTML 容器
- 返回 HTML 文件路径，前端可通过 `<iframe>` 嵌入预览

#### 5.4 在 PreviewController 中的调用

```csharp
// GET /api/sys/preview/{fileId}
public async Task<IActionResult> GetPreview(string fileId)
{
    var file = await _fileService.GetByIdAsync(fileId);
    var pdfPath = await _previewService.ConvertToPdfAsync(
        file.PhysicalPath,
        Path.Combine(_tempDir, file.Id + ".pdf"));
    return PhysicalFile(pdfPath, "application/pdf");
}
```

#### 5.5 注意事项

- LibreOffice 首次启动较慢，建议预热一个实例常驻后台。
- 并发请求下使用进程池 / 信号量限制 `soffice.exe` 实例数。
- 预览缓存：相同输入文件可缓存结果，用文件哈希 + 源路径作 Key。

```csharp
// ===== 文件上传完整流程（Controller 层）=====
[HttpPost("upload")]
public async Task<IActionResult> Upload(IFormFile file)

    using var stream = new FileStream(filePath, FileMode.Create);
    await file.CopyToAsync(stream);

    return Ok(new { FileId = fileId, FileName = safeName });
}

}

// ===== 视频截图 =====

Console.WriteLine($"时长: {info.Duration}, 分辨率: {info.Width}x{info.Height}");
await VideoUtil.GenerateThumbnailAsync("video.mp4", "screenshot.png", seconds: 5);

// ===== 图片压缩 =====
byte[] compressed = ImageUtil.Compress(rawBytes, maxWidth: 1920, quality: 80);
```

## ❓ 常见问题

**1. 问：文件上传后无法识别的常见原因？**
答：按顺序排查：文件大小是否超过限制 → 扩展名是否在白名单内 → 是否命中黑名单 → 磁盘空间是否充足 → 磁盘权限是否正确。

**2. 问：FFmpeg 找不到怎么办？**
答：Windows 下从官方下载 `ffmpeg-release-essentials.zip`，解压后 `bin/` 目录加入系统 PATH，或在 `appsettings.json` 中显式指定 `FFmpeg:BinaryPath`。Linux 使用 `apt-get install ffmpeg`。

**3. 问：如何防止路径遍历攻击？**
答：始终使用 `FileSecurityUtil.SanitizeFileName()` 清理用户上传的文件名，不要直接拼接路径；建议使用 GUID 前缀作为最终存储文件名。

**4. 问：大文件上传内存溢出？**
答：使用 `ChunkUploadService` 分片上传，默认 5MB 一片，支持断点续传；前端建议并发 3 片上传，全部完成后由服务端合并。

**5. 问：Office 文档预览失败？**
答：确认 LibreOffice 是否已安装并配置路径；首次启动较慢，建议预热常驻；并发请求使用进程池 + 信号量限制 `soffice.exe` 实例数。

**6. 问：上传到公开站点的图片会泄露 GPS 位置吗？**
答：会！务必使用 `ImageGeoUtil.ReadGeoInfo` 检测，或使用 `ImageUtil` 重新压缩图片（重编码会自动丢弃 EXIF 信息）。
