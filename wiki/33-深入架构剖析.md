## 🌱 组合根：应用启动流程

### 启动时序图

```
Program.cs / WebApplication.CreateBuilder()
  ├─ 配置 appsettings.json / 环境变量 / 命令行参数
  ├─ 添加 Controllers + 全局异常过滤器 (ApiExceptionFilter)
  ├─ 配置 Json 序列化（驼峰 + 空值忽略 + 枚举转字符串）
  ├─ 添加 Swagger 文档 + XML 注释集成
  ├─ 配置【JWT 认证】
  │    ├─ TokenValidationParameters（签名验证 / 有效期 / 颁发者 / 受众）
  │    └─ OnTokenValidated 事件中检查 Token 黑名单（实时吊销）
  ├─ 添加 CORS 策略（允许 localhost:5173/4173/3000，生产需配置实际域名）
  ├─ 添加速率限制 (100 req/min，队列深度 10)
  ├─ 注册 HttpContextAccessor
  ├─ 注册 ICurrentUser → CurrentUser 实现 (Scoped)
  ├─ 注册 ITenantContext → TenantContext 实现 (Scoped)
  ├─ 注册 SignalR（实时通知 Hub）
  ├─ 【模块程序集注入】
  │    ├─ 注入 Sys/Cms/Bpm/CodeGen/Tasks 五个模块的程序集
  │    └─ 供 JeeSiteDbContext 发现 EntityTypeConfiguration
  ├─ 【EF Core DbContext 配置】
  │    ├─ 根据 DatabaseProvider 配置连接字符串
  │    ├─ 添加 3 个拦截器：Audit / TreeEntity / SoftDelete / CorpEntity
  │    └─ 使用 provider factory 动态适配不同数据库
  ├─ 【FusionCache 二级缓存】
  │    ├─ 默认过期 30 分钟（L2）
  │    ├─ L1 内存缓存 + L2 Redis 分布式缓存
  │    └─ System.Text.Json 序列化
  ├─ 【模块自动加载】
  │    ├─ ModuleLoader 扫描 AppDomain 中所有 IModuleInstaller
  │    └─ 调用每个安装器的 ConfigureServices 注册服务
  ├─ 覆盖 Notification 服务（NullNotification → SignalRNotification 实现）
  ├─ Build WebApplication
  ├─ 【种子数据初始化】
  │    ├─ SeedData.InitializeAsync(app.Services)
  │    ├─ EnsureCreated 保证数据库存在
  │    └─ 幂等插入 admin/admin 及基础菜单数据
  ├─ 【Quartz 任务调度初始化】
  │    ├─ SchedulerService.InitDefaultJobsAsync
  │    └─ 启动默认任务
  ├─ 开发环境启用 Swagger / SwaggerUI
  ├─ 管线注册（顺序关键！）
  │    ├─ app.UseCors()
  │    ├─ app.UseRateLimiter()
  │    ├─ app.UseMiddleware<SecurityHeadersMiddleware>()
  │    ├─ app.UseAuthentication()
  │    ├─ app.UseAuthorization()
  │    ├─ app.UseMiddleware<RequestLogMiddleware>()
  │    ├─ app.UseMiddleware<TenantResolutionMiddleware>()
  │    └─ app.MapControllers() + MapHub<NotificationHub>
  └─ app.Run()
```

### 关键设计决策

| 决策点 | 选择 | 原因 |
|--------|------|------|
| 模块程序集显式注入 | 是 | 避免 JeeSiteDbContext 扫描所有程序集，提升模型构建性能 |
| 拦截器 Singleton 注入 | 是 | 无状态，可复用，减少 GC 压力 |
| SeedData 幂等初始化 | 是 | 首次启动自动创建 admin，开箱即用，多次启动不报错 |
| ModuleLoader 反射扫描 | 是 | 模块开发者不需要修改核心代码，只需实现 IModuleInstaller |
| Notification 覆盖注册 | 是 | Core 层默认 Null 实现，Web 层替换为 SignalR，符合依赖倒置 |

## 🛡️ 中间件管线：请求流转

### 管线执行顺序（顺序至关重要）

```
请求进入
  ↓
[CORS] 处理预检请求，设置 CORS 头
  ↓
[速率限制] 检查 IP 频率，超过限制返回 429
  ↓
[安全头] 注入 OWASP 推荐安全响应头
  ↓
[JWT 认证] 读取 Authorization 头，验证 Token，设置 User
  └── [Token 黑名单检查] 吊销被撤回的 Token（Redis / L1 缓存）
  ↓
[授权] 检查 [Authorize] / [Permission] 策略
  ↓
[请求日志] 记录请求方法、路径、状态码、耗时
  ↓
[租户解析] 解析 TenantCode 到 TenantContext (Scoped)
  ↓
[MVC] 路由匹配 → Controller → Action（业务逻辑）
  ↓
响应返回
```

### 各中间件职责详解

#### 1. CORS 中间件

```csharp
services.AddCors(options => {
    options.AddPolicy("AllowFrontend", policy => {
        policy.WithOrigins("http://localhost:5173", "http://localhost:3000")
              .AllowAnyHeader()
              .AllowAnyMethod()
              .AllowCredentials();
    });
});
```

**注意**: 必须注册在**最前**，用于处理 `OPTIONS` 预检请求。生产环境应严格限制允许的 Origin 列表。

#### 2. 速率限制中间件

```csharp
services.AddRateLimiter(options => {
    options.AddFixedWindowLimiter("Global", config => {
        config.PermitLimit = 100;
        config.Window = TimeSpan.FromMinutes(1);
        config.QueueLimit = 10;
    });
});
```

**防护目标**: 防止暴力破解密码、爬虫滥用、简单 DDoS 攻击。

#### 3. 安全头中间件（SecurityHeadersMiddleware）

注入以下 OWASP 推荐安全响应头：

| 头 | 值 | 目的 |
|----|-----|------|
| `X-Content-Type-Options` | `nosniff` | 防止 MIME 类型嗅探 |
| `X-Frame-Options` | `SAMEORIGIN` | 防止 ClickJacking / iframe 嵌入攻击 |
| `X-XSS-Protection` | `1; mode=block` | 启用浏览器内置 XSS 防护 |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | 控制 Referrer 信息泄漏 |
| `Strict-Transport-Security` | `max-age=31536000; includeSubDomains` | HTTPS 强制（仅 HTTPS 时发送） |
| `Content-Security-Policy` | `default-src 'self'` | 资源白名单，防 XSS 载荷执行 |

#### 4. 请求日志中间件（RequestLogMiddleware）

记录信息：请求方法 + 路径 + 响应状态码 + 处理耗时 + 用户编码（已认证时）。

输出到**结构化日志**，可接入 Loki/Grafana/ELK 做集中分析。

#### 5. 租户解析中间件（TenantResolutionMiddleware）

**唯一职责**: 将租户编码解析到 `TenantContext`（Scoped）。

解析优先级：

```
1. HTTP Header: X-Tenant-Code
   ↓
2. 子域名提取: tenant1.example.com → tenant1
   ↓
3. JWT Claim: tenant_code (用户登录后携带)
   ↓
4. 保持 null（单租户模式不受影响）
```

## 🏢 多租户架构：租户解析与隔离

### 租户数据隔离模型

JeeSite.NET 采用**共享数据库 + 行级隔离**策略：

```
┌────────────────────────────────────────────────────────────┐
│           共享数据库 (Single Database / Shared Schema)       │
├────────────────────────────────────────────────────────────┤
│ Tenant A ┌──────────┐  Tenant B ┌──────────┐                 │
│         │  Data    │            │  Data    │                 │
│         │ Row 1..N│            │ Row 1..N│                 │
│         └──────────┘            └──────────┘                 │
└────────────────────────────────────────────────────────────┘
         ↓ 过滤条件
CorpEntityInterceptor 自动追加: WHERE CorpCode = @CurrentTenant
```

**优点**:
- 管理简单，无需维护多个数据库
- 成本低，共享连接池
- 租户增加无需变更数据库结构

**缺点**:
- 数据量极大时需考虑分库分表（当前版本可选扩展）

### 代码中的关键协作组件

| 组件 | 位置 | 职责 |
|------|------|------|
| `TenantContext` | `JeeSiteNET.Core` | Scoped 容器，存储当前请求租户 |
| `TenantResolutionMiddleware` | `Web.Api/Middleware` | 解析租户编码到上下文 |
| `ICorpEntity` | `JeeSiteNET.Core` | 实体接口，包含 `CorpCode` |
| `CorpEntityInterceptor` | `Infrastructure/Interceptors` | EF Core 查询自动过滤 |

### 如何开启多租户

```json
{
  "MultiTenancy": { "Enabled": true }
}
```

所有实现已就绪，无需修改代码。开启后：
- 每个用户登录后自动归属其所属公司（Corp）
- 查询自动过滤 `corp_code = 当前用户 corp_code`
- 管理员可通过顶部导航"切换租户"查看不同公司数据

## 💾 多数据库支持：Provider 工厂模式

### 支持的数据库

| 数据库 | Provider | NuGet 包 | 说明 |
|--------|----------|----------|------|
| SQL Server | `SqlServer` | Microsoft.EntityFrameworkCore.SqlServer | 默认，开箱即用 |
| SQLite | `Sqlite` | Microsoft.EntityFrameworkCore.Sqlite | 开发零配置 |
| PostgreSQL | `PostgreSQL` | Npgsql.EntityFrameworkCore.PostgreSQL | 需手动安装 |
| 达梦 DM8 | `Dm` | DmProvider + Dm.EntityFrameworkCore | 国产信创 |
| 人大金仓 | `KingbaseES` | Kdbndp.EntityFrameworkCore.KingbaseES | 国产信创 |

### 配置切换

只需修改 `appsettings.json`，**无需重新编译**：

```json
{
  "DatabaseProvider": "PostgreSQL",
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Database=JeeSiteNET;Username=postgres;Password=pwd"
  }
}
```

### 工厂模式架构

```csharp
// DatabaseProviderFactory.cs
public static void UseProvider(
    this DbContextOptionsBuilder options,
    DatabaseProviderType provider,
    string connectionString)
{
    switch (provider)
    {
        case SqlServer:
            options.UseSqlServer(connectionString, b => b.CommandTimeout(30));
            break;
        case Sqlite:
            options.UseSqlite(connectionString);
            break;
        case PostgreSQL:
            options.UseNpgsql(connectionString);
            break;
        default:
            UseExternalProvider(options, provider, connectionString);
            break;
    }
}

// 外部数据库（国产）通过反射动态绑定
private static void UseExternalProvider(...)
{
    // 反射从外部 NuGet 包加载 UseXxx 扩展方法
    // 不依赖具体包，支持按需安装
}
```

**设计亮点**:
- 核心项目**不依赖**国产数据库驱动，按需安装
- 反射动态绑定，**不需要**条件编译 `#if`
- 新增数据库支持只需修改工厂方法，不影响调用方

## 🛡️ 安全纵深防御：五层防护模型

JeeSite.NET 采用**纵深防御（Defense in Depth）**原则，五层防护确保应用安全：

```
┌─────────────────────────────────────────────────────────────┐
│  第一层：网络层 / CORS · Origin 白名单                        │
│  (限制哪些前端域名可访问 API，阻止跨域滥用)                    │
├─────────────────────────────────────────────────────────────┤
│  第二层：速率限制 / 单 IP 100 req/min + 队列 10               │
│  (防止暴力破解、爬虫、简单 DDoS)                              │
├─────────────────────────────────────────────────────────────┤
│  第三层：安全头 / OWASP 标准安全响应头 (X-Frame-Options 等) │
│  (浏览器级防护，抵御 ClickJacking / MIME Sniffing / XSS)     │
├─────────────────────────────────────────────────────────────┤
│  第四层：认证 / JWT + Token 黑名单（实时吊销）                 │
│  (谁可以访问系统；支持多设备登录控制、强制下线)                │
├─────────────────────────────────────────────────────────────┤
│  第五层：授权 / RBAC 权限 + 数据权限 + 字段权限               │
│  (用户能看什么、操作什么；行级 / 列级精细控制)                 │
└─────────────────────────────────────────────────────────────┘
```

### 第四层：JWT 认证 + Token 吊销（黑名单机制）

```csharp
options.Events = new JwtBearerEvents
{
    OnTokenValidated = async ctx =>
    {
        var cache = ctx.RequestServices.GetRequiredService<IFusionCache>();
        var tokenStr = ctx.Request.Headers["Authorization"]
                            .ToString().Replace("Bearer ", string.Empty);

        var blacklisted = await cache.GetOrDefaultAsync<string>(
            $"TokenBlacklist:{tokenStr}");

        if (!string.IsNullOrEmpty(blacklisted))
            ctx.Fail("Token has been revoked");
    }
};
```

**工作流程**:
1. 用户在设备 B 登录 → 设备 A 的 Token 加入黑名单（缓存键）
2. 每次请求验证 Token 有效性时检查黑名单（L1 内存 + L2 Redis）
3. 黑名单条目自动过期（和 Token 有效期一致，约 12 小时后自动清理）

这是**无数据库**方案，利用 FusionCache 二级缓存实现，性能优异。

**管理员操作**: 进入"系统管理 → 在线用户"，点击"踢出"即可实时吊销指定 Token。

### 第五层：RBAC + 细粒度授权

| 层级 | 控制粒度 | 说明 |
|------|---------|------|
| 菜单级 | `[Permission("sys:user:view")]` | 路由/页面级权限 |
| 按钮级 | `v-permission="'sys:user:add'"` | 前端按钮显示/隐藏 |
| 数据权限 | `DataScopeService.ApplyDataScope()` | 行级过滤（全部/本人/本部门/自定义 SQL） |
| 字段权限 | `FieldScopeService` | 列级：字段隐藏、只读、脱敏 |

### 密码安全

- BCrypt 哈希存储，加盐
- 支持密码强度校验（长度/大小写/数字/符号组合）
- 密码修改历史（防止循环使用最近 5 个历史密码）
- 定期强制修改提示

## 🧩 依赖注入生命周期：服务注册模式

### 生命周期选择原则

| 服务类型 | 生命周期 | 示例 |
|----------|---------|------|
| 无状态工具 | Singleton | `EncryptUtil`, `ExcelService`, `CaptchaUtil` |
| 仓储 | Scoped | `UserRepository`, `ArticleRepository` |
| 应用服务 | Scoped | `UserService`, `AuthService`, `ArticleService` |
| 当前用户/租户 | Scoped | `CurrentUser`, `TenantContext` |
| EF Core DbContext | Scoped | `JeeSiteDbContext` |
| EF Core 拦截器 | Singleton | `AuditInterceptor`, `SoftDeleteInterceptor` |
| FusionCache | Singleton | `IFusionCache`（内部管理） |
| SignalR Hub | Transient | `NotificationHub`（由框架管理） |

### 模块自动注册机制

```csharp
// ModuleLoader.cs
public void LoadModules(IServiceCollection services, IConfiguration configuration)
{
    var assemblies = AppDomain.CurrentDomain.GetAssemblies();

    foreach (var assembly in assemblies)
    {
        var installerTypes = assembly.GetTypes()
            .Where(t => typeof(IModuleInstaller).IsAssignableFrom(t)
                     && !t.IsInterface);

        foreach (var type in installerTypes)
        {
            var installer = (IModuleInstaller)Activator.CreateInstance(type)!;
            installer.ConfigureServices(services, configuration);
        }
    }
}
```

**优点**:
- 添加新模块只需新建项目 + 实现 `IModuleInstaller`
- 核心代码无需修改
- 符合**开闭原则（Open/Closed Principle）**

### 模块程序集与 EF Core 配置发现

```csharp
builder.Services.AddSingleton<IEnumerable<Assembly>>(new[]
{
    typeof(User).Assembly,             // Sys 模块
    typeof(Article).Assembly,          // Cms 模块
    typeof(ApprovalRecord).Assembly,   // Bpm 模块
    typeof(GenTable).Assembly,         // CodeGen 模块
    typeof(SysJob).Assembly,           // Tasks 模块
});
```

```csharp
// JeeSiteDbContext.cs
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    foreach (var assembly in _configurationAssemblies)
    {
        modelBuilder.ApplyConfigurationsFromAssembly(assembly);
    }
}
```

每个模块在自己的程序集中存放 `IEntityTypeConfiguration<T>`，`JeeSiteDbContext` 批量应用。

**优点**:
- 模块内聚，配置和实体在一起
- 核心 `JeeSiteDbContext` 不需要随模块增加修改
- 性能比自动扫描所有程序集更好
