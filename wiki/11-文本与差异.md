### 一、DiffMatchPatch

`DiffMatchPatch`（`Utils/DiffMatchPatch.cs`）是 Google **Diff-Match-Patch** 算法的 C# 实现，用于两段文本的差异比较、补丁应用以及相似度计算。

#### 1.1 核心数据结构

```csharp
public enum Operation { Equal, Insert, Delete }

public class Diff
{
    public Operation Operation { get; set; }
    public string Text { get; set; }
}

public class Patch
{
    public List<Diff> Diffs { get; set; }
    public int Start1 { get; set; }
    public int Start2 { get; set; }
    public int Length1 { get; set; }
    public int Length2 { get; set; }
}
```

#### 1.2 核心方法

**（1）差异计算**
```csharp
public List<Diff> ComputeDiffs(string oldText, string newText)
public List<Diff> ComputeDiffsWithTimeout(
    string oldText,
    string newText,
    double timeoutSeconds = 1.0)
```
- `ComputeDiffs`: 计算 `oldText` 与 `newText` 的差异列表，每个 `Diff` 标注 `Equal / Insert / Delete`。
- `ComputeDiffsWithTimeout`: 带超时保护版本；大文本（> 1MB）下避免卡死。超时后返回一个简化的差异（整段文本被整体标记为 Delete + Insert）。

**（2）差异 → HTML 高亮**
```csharp
public string DiffToHtml(string oldText, string newText)
```
输出 HTML：`<del>` 标记删除、`<ins>` 标记插入、普通文本原样输出。CSS 样式：

```css
del { background: #ffe6e6; text-decoration: line-through; }
ins { background: #e6ffe6; text-decoration: none; }
```

**（3）补丁生成与应用**
```csharp
public List<Patch> MakePatches(string oldText, string newText)
public (string newText, bool[] results) ApplyPatches(
    string text,
    List<Patch> patches)
```
- `MakePatches`: 基于差异生成补丁列表。
- `ApplyPatches`: 将补丁应用到目标文本。`results[i]` 表示第 i 个补丁是否成功应用。

**（4）编辑距离**
```csharp
public int LevenshteinDistance(string a, string b)
```
返回从 `a` 到 `b` 所需的最少单字符编辑（插入 / 删除 / 替换）次数。

**（5）文本相似度**
```csharp
public double TextSimilarity(string a, string b)
```
返回 `[0.0, 1.0]` 的相似度，基于 `1.0 - Levenshtein / Max(a.Length, b.Length)`。

#### 1.3 示例

```csharp
var dmp = new DiffMatchPatch();

// 1. 差异计算
var diffs = dmp.ComputeDiffs(
    "今天天气很好，适合出门散步。",
    "今天天气很棒，适合出门跑步。");
// [ Equal("今天天气很"), Insert("棒"), Delete("好"),
//   Equal("，适合出门"), Insert("跑"), Delete("散"), Equal("步。") ]

// 2. HTML 输出
string html = dmp.DiffToHtml(oldText, newText);
// → "...今天天气很<ins>棒</ins><del>好</del>，适合出门<ins>跑</ins><del>散</del>步。..."

// 3. 补丁流程
var patches = dmp.MakePatches(oldText, newText);
var (recovered, results) = dmp.ApplyPatches(oldText, patches);
// recovered == newText

// 4. 相似度
double sim = dmp.TextSimilarity("hello world", "hello word");
// sim ≈ 0.91
```

#### 1.4 典型业务场景

- **Wiki / CMS 内容对比**：对比同一篇文章的两个版本。
- **配置文件变更追踪**：对比两次部署时 `appsettings.json` 的差异。
- **合同文本修订历史**：法律/合同系统中留存修订记录。
- **答题相似度检测**：主观题批改辅助。

### 二、PinyinUtil

`PinyinUtil`（`Utils/PinyinUtil.cs`）用于中文 → 拼音转换，基于内嵌拼音字典。

#### 2.1 核心方法

**（1）带声调拼音**
```csharp
public static string GetPinyin(string chineseText)
```
示例：
```csharp
PinyinUtil.GetPinyin("你好");     // "nǐ hǎo"
PinyinUtil.GetPinyin("重庆");     // "chóng qìng"
```

**（2）无声调拼音**
```csharp
public static string GetPinyinWithoutTone(string chineseText)
```
示例：
```csharp
PinyinUtil.GetPinyinWithoutTone("你好");  // "ni hao"
```

**（3）首字母大写**
```csharp
public static string GetFirstLetter(string chineseText)
```
示例：
```csharp
PinyinUtil.GetFirstLetter("张三");   // "ZS"
PinyinUtil.GetFirstLetter("李四");   // "LS"
```

**（4）全拼连写**
```csharp
public static string GetFullPinyin(string chineseText)
```
示例：
```csharp
PinyinUtil.GetFullPinyin("张三");   // "zhangsan"
```

#### 2.2 多音字处理说明

- 字典中内置常见多音字的首选读音（如"重"优先读 `chóng`）。
- 对于需要精确控制的场景（人名、地名），建议使用 `PinyinUtil` 作为候选结果生成器，再由用户手动确认。

#### 2.3 典型业务场景

| 场景 | 使用方法 | 示例 |
|------|---------|------|
| 联系人按拼音排序 | `GetPinyinWithoutTone` | 对 `List<User>` 按拼音字段排序 |
| 搜索 / 筛选 | `GetFirstLetter` | 在搜索框输入 `zs` 匹配"张三" |
| URL Slug | `GetFullPinyin` | `/article/ni-hao-shi-jie` |
| 导出 Excel 附带拼音 | `GetPinyin` | 在中文列旁加一列拼音 |

### 三、IdcardUtil

`IdcardUtil`（`Utils/IdcardUtil.cs`）对中国第二代居民身份证号码（18 位）进行合法性校验与信息提取。

#### 3.1 18 位身份证结构

```text
位置  含义             示例
1-6   行政区划代码       110101   （北京市东城区）
7-14  出生日期（YYYYMMDD） 19900101
15-17 顺序号（奇数=男，偶数=女）
18    校验码            0-9 / X
```

校验算法（ISO 7064:1983.MOD 11-2）：
1. 前 17 位分别乘以权重 `[7,9,10,5,8,4,2,1,6,3,7,9,10,5,8,4,2]`。
2. 将乘积求和并对 11 取模。
3. 在表 `['1','0','X','9','8','7','6','5','4','3','2']` 中按模取值得到第 18 位。

#### 3.2 核心方法

**（1）综合验证**
```csharp
public static bool Validate(string idcardNumber)
```
依次执行：
- 长度 = 18，且只含数字（最后一位可为 X）
- 前 6 位行政区划代码有效（从内置表查询）
- 中间 8 位是合法日期（1900-01-01 起）且不晚于今天
- 校验位通过 ISO 7064 MOD 11-2 算法

**（2）提取出生日期**
```csharp
public static DateTime? GetBirthDate(string idcardNumber)
```

**（3）提取性别**
```csharp
public static Gender GetGender(string idcardNumber)  // Gender.Male / Gender.Female
```

**（4）提取行政区划代码**
```csharp
public static string GetRegionCode(string idcardNumber)
```

**（5）计算当前年龄**
```csharp
public static int GetAge(string idcardNumber)
```

#### 3.3 示例

```csharp
const string id = "11010119900101123X";

bool valid = IdcardUtil.Validate(id);        // true
DateTime? birth = IdcardUtil.GetBirthDate(id);// 1990-01-01
Gender g = IdcardUtil.GetGender(id);          // Gender.Male
string region = IdcardUtil.GetRegionCode(id); // "110101"
int age = IdcardUtil.GetAge(id);              // 取决于当前日期
```

#### 3.4 使用建议

- 前端可调用 `Validate` 做基本格式校验；若要验证真实性，需对接公安部接口。
- 身份证号属于个人敏感信息，落地数据库必须使用 SM4 加密（见《加密与国密工具》章）。
- 15 位老身份证自动升级为 18 位（`idcardUtil.Upgrade15To18(id)`）后再做校验。

### 最小工作示例

```csharp
// ===== 拼音转换 =====
string pinyin = PinyinUtil.GetPinyin("中华人民共和国");
Console.WriteLine(pinyin);            // "zhōng huá rén mín gòng hé guó"

string initials = PinyinUtil.GetFirstLetter("张三");
Console.WriteLine(initials);          // "ZS"

string slug = "C# 高级编程".ToSlug();
Console.WriteLine(slug);              // "c-gao-ji-bian-cheng"

// ===== 身份证验证 =====
string idCard = "11010119900101123X";
bool isValid = IdcardUtil.Validate(idCard);
if (isValid)
{
    DateTime? birth = IdcardUtil.GetBirthDate(idCard);
    var gender = IdcardUtil.GetGender(idCard);
    int age = IdcardUtil.GetAge(idCard);
    string region = IdcardUtil.GetRegionCode(idCard);
    Console.WriteLine($"出生: {birth:yyyy-MM-dd}, 性别: {gender}, 年龄: {age}, 区域: {region}");
}

// ===== 文本差异比对 =====
var dmp = new DiffMatchPatch();
string oldText = "Hello, World!";
string newText = "Hello, JeeSite.NET!";
var diffs = dmp.ComputeDiffs(oldText, newText);
foreach (var diff in diffs)
{
    Console.WriteLine($"[{diff.Operation}] {diff.Text}");
}

// ===== 生成并应用补丁 =====
var patches = dmp.MakePatches(oldText, newText);
var (applied, _) = dmp.ApplyPatches(oldText, patches);

// ===== 脱敏显示 =====
Console.WriteLine("13812345678".MaskMobile());    // "138****5678"
Console.WriteLine("user@example.com".MaskEmail()); // "u***@example.com"

// ===== 友好时间 =====
Console.WriteLine(DateTime.Now.AddMinutes(-5).ToFriendlyString());  // "5 分钟前"
```

## ❓ 常见问题

**1. 问：拼音转换对生僻字处理如何？**
答：PinyinUtil 基于内置字库，对常用中文字符支持良好；极少数罕见字可能返回空字符串，建议前端做 fallback 显示原字符。

**2. 问：身份证验证支持 15 位老身份证吗？**
答：支持，`IdcardUtil.Upgrade15To18(id)` 会先补全年份和校验位，再按 18 位标准校验。

**3. 问：DiffMatchPatch 的补丁可以跨版本应用吗？**
答：补丁是基于原始文本计算的，必须保证补丁来源文本一致，否则可能无法正确应用；建议使用 `ApplyPatches` 返回的成功标志判断。

**4. 问：编辑距离和文本相似度有什么区别？**
答：编辑距离 = 需要多少次单字符操作（插入/删除/替换）才能从 a 变到 b；相似度 = 1 - (距离 / 最大长度)，范围 [0, 1]。

**5. 问：ToSlug 中文怎么处理？**
答：中文会被转换为无声调全拼连写，再将非 ASCII 和非字母数字替换为 `-`，多个 `-` 合并为一个，首尾去除。
