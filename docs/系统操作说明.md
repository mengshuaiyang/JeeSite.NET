## 一、系统概述

### 1.1 系统简介

**JeeSite.NET** 是基于 .NET 10 / ASP.NET Core / EF Core / Vue 3 / Ant Design Vue 构建的企业级快速开发平台。它提供完整的企业级后台管理功能，包括权限管理、内容管理、代码生成、任务调度、工作流、AI 智能问答等核心能力。

### 1.2 技术架构

```
┌────────────────────────────────────┐
│ 前端（Vue 3 + Vite + Pinia）        │
│  Ant Design Vue 4 + TypeScript     │
└────────────────┬───────────────────┘
                 │ HTTP / SignalR / WebSocket
┌────────────────▼───────────────────┐
│ API层（ASP.NET Core）              │
│  统一响应 + Swagger + JWT          │
└────────────────┬───────────────────┘
                 │ 应用服务 / 仓储
┌────────────────▼───────────────────┐
│ 业务模块（Sys / CMS / BPM ...）    │
│  模块化架构 · 领域驱动             │
└────────────────┬───────────────────┘
                 │ EF Core + FusionCache
┌────────────────▼───────────────────┐
│ 数据层 / 基础设施                  │
│  SQL Server · Redis · MinIO · ES   │
└────────────────────────────────────┘
```

### 1.3 功能模块一览

| 模块 | 主要功能 | 适用对象 |
|------|---------|---------|
| **Sys（系统管理）** | 用户/角色/菜单/机构/公司/岗位/区域/字典/配置/文件/消息/日志/审计/缓存/监控 | 系统管理员 |
| **CMS（内容管理）** | 站点/栏目/文章/评论/留言/标签/举报/访问统计/AI 智能问答 | 内容编辑、运营 |
| **CodeGen（代码生成）** | 表导入/字段配置/模板管理/预览/下载 ZIP | 开发人员 |
| **Tasks（任务调度）** | Quartz.NET 任务管理/手动执行/日志查看 | 运维人员 |
| **BPM（工作流）** | 请假审批流程/审批记录/我的审批 | 全体员工 |
| **App（移动端）** | 意见反馈/版本升级管理 | 移动端运营 |
| **Test（示例模块）** | 树形数据/表格数据示例 | 开发参考 |

### 1.4 系统默认账号

| 账号 | 密码 | 权限 | 说明 |
|------|------|------|------|
| `admin` | `admin` | 超级管理员 | 拥有所有系统权限，生产环境请立即修改 |

## 二、快速入门

### 2.1 启动系统

根据部署方式不同选择对应的操作：

#### 🚀 开发环境

```bash
# 后端
cd d:\Projects\jeesite.net
dotnet run --project src/JeeSiteNET.Web.Api
# API: http://localhost:5000
# Swagger: http://localhost:5000/swagger

# 前端（另开终端）
cd d:\Projects\jeesite.net\frontend
pnpm install   # 首次需要
pnpm dev
# 前端: http://localhost:5173
```

#### 🐳 Docker 部署

```bash
cd d:\Projects\jeesite.net
docker compose up -d

# 等所有服务就绪后访问
# 前端: http://localhost:3000
# API:  http://localhost:5000
# Swagger: http://localhost:5000/swagger
```

### 2.2 首次登录

1. 打开浏览器访问系统首页（开发环境: http://localhost:5173）
2. 在登录页面输入账号密码：`admin / admin`
3. 首次登录后，系统会自动跳转到 Dashboard 首页

> 💡 **重要提示**: 生产环境部署后请立即修改 admin 密码，并限制管理后台的访问来源。

### 2.3 界面布局

```
┌─────────────────────────────────────────────────────────────┐
│ [Logo] JeeSite.NET    [消息🔔] [用户信息 ▾] [切换租户▾]     │
├────────┬────────────────────────────────────────────────────┤
│        │ 首页 系统管理 ▾ 内容管理 ▾ 代码生成 ▾ 任务调度 ▾ ...│
│ 菜单   ├────────────────────────────────────────────────────┤
│ 导航   │                                            │
│ (左侧) │   主内容区                                   │
│        │                                            │
│        │                                            │
└────────┴────────────────────────────────────────────────────┘
```

- **顶部导航栏**: Logo / 站点切换 / 消息中心 / 用户菜单
- **左侧菜单**: 动态根据用户权限展示可操作的模块菜单
- **内容区**: 业务操作的主要页面

## 三、系统管理模块（Sys）

### 3.1 用户管理

#### 操作路径

`系统管理 → 用户管理`

#### 功能说明

- **新增用户**: 点击"新增"按钮，填写登录账号、姓名、密码、归属公司/机构/岗位、角色
- **编辑用户**: 在用户列表中点击"编辑"，可修改用户基本信息、角色、状态
- **删除用户**: 选中用户后点击"删除"（建议使用禁用而非删除）
- **启用/禁用**: 修改用户状态字段
- **重置密码**: 管理员可强制重置用户密码
- **数据权限**: 通过"角色"间接配置，用户的权限由其所拥有的角色决定

#### 字段说明

| 字段 | 说明 | 必填 |
|------|------|------|
| 登录账号 | 用户登录的唯一标识，系统内唯一 | ✅ |
| 用户姓名 | 用户真实姓名 | ✅ |
| 密码 | 首次设置需要符合密码强度要求（字母+数字+符号，8位以上） | ✅ |
| 归属公司 | 用户所属的公司（多租户） | ✅ |
| 归属机构 | 用户所属的部门/机构 | ✅ |
| 归属岗位 | 用户在公司内的岗位 | ⬜ |
| 角色列表 | 用户拥有的角色（权限由此决定） | ✅ |
| 手机号 | 接收短信验证码 / 短信登录 | ⬜ |
| 邮箱 | 接收邮件验证码 / 邮件登录 | ⬜ |
| 状态 | 0=正常，1=禁用 | ✅ |

### 3.2 角色管理

#### 操作路径

`系统管理 → 角色管理`

#### 功能说明

角色是权限的载体，用户通过拥有角色获得权限。

- **新增角色**: 设置角色名称、角色编码、排序
- **分配菜单权限**: 在角色详情中设置该角色可以访问的菜单
- **分配数据权限**: 设置该角色可以看到的**数据范围**
  - **全部数据**: 不受限制
  - **本公司数据**: 仅查看本公司的数据
  - **本公司及以下**: 本公司+所有子公司的数据
  - **本机构数据**: 仅查看所属机构的数据
  - **本机构及以下**: 本机构+所有子机构的数据
  - **仅本人数据**: 仅查看自己创建/归属的数据
- **分配字段权限**: 对某些敏感字段（如金额、手机号）控制是否可见
- **绑定用户**: 在角色页面查看/管理拥有该角色的用户列表

#### 推荐角色划分建议

| 角色 | 典型权限 | 数据范围 |
|------|---------|---------|
| 超级管理员 | 所有权限 | 全部数据 |
| 系统管理员 | 用户/角色/机构/配置/监控 | 本公司数据 |
| 内容编辑 | 文章/栏目/评论管理 | 本机构数据 |
| 普通员工 | 个人工作台/文档查看 | 仅本人 |

### 3.3 菜单管理

#### 操作路径

`系统管理 → 菜单管理`

#### 功能说明

菜单是系统功能的导航骨架，每个菜单项对应一个功能页面或动作。

- **新增菜单**: 树形结构支持多层嵌套
  - **类型**: 菜单（页面）/ 按钮（操作权限）
  - **路径**: 前端路由路径
  - **权限标识**: 用于前端按钮级权限判断（如 `sys:user:add`）
- **菜单树形维护**: 支持拖拽排序、上移下移
- **模块菜单隔离**: 支持按 sys 代码区分不同业务系统的菜单

#### 权限标识规范

```
{模块}:{实体}:{操作}
例: sys:user:add       (用户管理-新增)
    sys:user:edit      (用户管理-编辑)
    sys:user:delete    (用户管理-删除)
    cms:article:view   (CMS-文章查看)
```

### 3.4 机构管理

#### 操作路径

`系统管理 → 机构管理`

#### 功能说明

机构以树形结构组织，支持任意层级的部门/子部门。

- 新增机构: 设置机构名称、编码、上级机构、类型（公司/部门/小组）
- 查看机构下员工: 在机构详情中查看归属员工
- 机构树维护: 支持拖拽调整层级和排序

### 3.5 公司管理

#### 操作路径

`系统管理 → 公司管理`

#### 功能说明

公司是多租户的边界，不同公司的数据天然隔离。

- 新增公司: 设置公司名称、编码、联系信息
- **切换当前公司**: 顶部导航栏的"站点/租户切换"下拉
- 公司下机构维护: 与机构管理联动
- 数据隔离: 不同公司的用户、文章、配置等数据完全隔离

### 3.6 岗位管理

#### 操作路径

`系统管理 → 岗位管理`

#### 功能说明

岗位是组织架构中的职位概念，用于归类员工身份。

- 新增岗位: 岗位名称 + 岗位编码 + 排序
- 与角色关系: 岗位可绑定角色，用户拥有岗位即继承其权限

### 3.7 区域管理

#### 操作路径

`系统管理 → 区域管理`

#### 功能说明

预置省/市/区三级行政区划树形结构，供业务功能选择地区使用。

- 系统初始化已内置常见行政区划数据
- 支持手动新增自定义区域
- 提供 REST API 供其他功能调用

### 3.8 字典管理

#### 操作路径

`系统管理 → 字典类型` → `字典数据`

#### 功能说明

字典用于集中管理系统中所有枚举值（如下拉框选项、状态值）。

**字典类型**（定义字典分类）：
- 如: 用户状态（`sys_user_status`）、性别（`sys_user_sex`）等

**字典数据**（具体选项）：
- 每一条字典数据包含: 标签名、值、排序、样式
- 前端/后端统一通过字典类型获取枚举列表

#### 常用字典示例

| 字典类型 | 字典值说明 |
|---------|-----------|
| `sys_user_status` | 0=正常, 1=禁用 |
| `sys_show_hide` | 0=隐藏, 1=显示 |
| `sys_yes_no` | 0=否, 1=是 |
| `cms_article_status` | 0=草稿, 1=已发布, 2=下架 |

### 3.9 配置管理

#### 操作路径

`系统管理 → 系统配置`

#### 功能说明

用于管理系统级参数，读取通过配置服务统一访问。

- 系统名称、Logo URL、备案号
- 邮件发送配置（SMTP 主机/端口/用户名/密码）
- 短信服务配置（接口地址 / API Key）
- 文件上传大小限制
- 默认密码策略

#### 配置项示例

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `sys.name` | JeeSite.NET | 系统显示名称 |
| `sys.logo` | `/logo.png` | Logo URL 路径 |
| `upload.maxSize` | `10485760` | 单文件最大 10MB |
| `pwd.minLength` | `8` | 密码最小长度 |
| `pwd.needDigit` | `true` | 密码必须包含数字 |

### 3.10 模块管理

#### 操作路径

`系统管理 → 模块管理`

#### 功能说明

管理已安装的业务模块（Sys / CMS / BPM / CodeGen / Tasks / App 等）。

- 查看模块版本、安装状态
- 模块启用/禁用开关
- 查看模块依赖关系（如 BPM 依赖 Sys）

### 3.11 消息管理

#### 操作路径

`系统管理 → 消息中心 → 收件箱 / 发件箱 / 模板 / 推送`

#### 功能说明

**站内消息**（收件/发件）：
- 系统内部消息通知，支持按用户/角色/机构定向发送
- 支持已读/未读状态，顶部导航栏的 🔔 图标实时提醒
- SignalR 实时推送，无需刷新页面

**消息模板**：
- 可预置消息模板（如审批通知、任务提醒、欢迎消息）
- 模板支持占位符变量（`{userName}` / `{date}` 等）

**消息推送**：
- 管理员可发起系统级推送（全体/指定角色/指定机构）
- 查看推送记录与送达统计
- 支持定时推送

### 3.12 文件管理

#### 操作路径

`系统管理 → 文件管理`

#### 功能说明

统一文件管理中心，支持上传/预览/下载。

- **上传方式**: 拖拽上传 / 分片上传 / 大文件断点续传
- **存储后端**: 本地存储 / MinIO / Aliyun OSS / S3（在配置中指定）
- **文件预览**: 图片/文本/PDF 直接预览；Office 文档通过 LibreOffice 转换
- **权限控制**: 文件归属 + 是否公开访问标记
- **垃圾文件清理**: 定期清理超过有效期的临时文件

#### 上传安全策略

| 校验类型 | 说明 |
|---------|------|
| 扩展名白名单 | 仅允许 `.jpg/.png/.pdf/.doc/.docx/.xls/.xlsx` 等安全类型 |
| 文件签名校验 | 读取文件头（Magic Number）判断真实格式是否与扩展名匹配，防止伪装 |
| 路径遍历防御 | 文件名中 `../` 等被自动清洗，防止覆盖系统文件 |
| 文件大小限制 | 由配置项 `upload.maxSize` 控制 |

### 3.13 国际化（多语言）

#### 操作路径

`系统管理 → 语言管理`

#### 功能说明

- 维护系统界面的多语言字典
- 支持 zh-CN / en-US 等语言版本，可自由扩展
- 通过 Lang API 统一读取

### 3.14 在线用户

#### 操作路径

`系统管理 → 在线用户`

#### 功能说明

查看当前在线用户列表。

- 登录时间、IP、浏览器、设备信息
- **强制下线**: 选中用户后点击"踢出"，其 JWT Token 立即失效

### 3.15 缓存管理

#### 操作路径

`系统管理 → 缓存管理`

#### 功能说明

查看并清理系统缓存（基于 FusionCache，两层缓存: L1 内存 + L2 Redis）。

- 查看缓存键数量、命中率、内存占用
- **按模块清理**: 如"清理菜单缓存"、"清理字典缓存"
- **全局清理**: 清空所有缓存（⚠️ 会导致性能短暂下降）

### 3.16 日志管理

#### 操作路径

`系统管理 → 日志管理`

#### 功能说明

记录用户在系统中的所有关键操作日志。

- 日志类型: 登录/退出 / 新增 / 修改 / 删除 / 导入 / 导出 / 异常
- 可按时间、用户、类型、IP、关键字检索
- 支持日志归档（超过 30 天自动转归档库）

#### 日志字段

| 字段 | 说明 |
|------|------|
| 日志标题 | 如"用户登录成功"、"删除用户 admin" |
| 请求路径 | /api/v1/sys/auth/login |
| 方法 | POST / GET / PUT / DELETE |
| 请求参数 | JSON 格式的请求体 |
| 响应时长 | 毫秒 |
| 操作用户 | 操作人账号 + 姓名 |
| 客户端 IP | 来源 IP + 地理位置解析 |
| User Agent | 浏览器 / 设备 / 操作系统 |
| 异常信息 | 若有错误，记录堆栈 |

### 3.17 审计跟踪

#### 操作路径

`系统管理 → 审计记录`

#### 功能说明

审计跟踪是比操作日志更细粒度的**数据变更记录**。

- 记录实体级别的每一次 Create / Update / Delete
- 可查看变更前后的字段对比（如: `status: 0 → 1`）
- 支持按时间、实体类型、操作人过滤检索
- 满足合规审计需求

### 3.18 系统监控

#### 操作路径

`系统管理 → 系统监控`

#### 功能说明

实时监控服务器运行状态。

| 指标 | 说明 |
|------|------|
| CPU 使用率 | 实时折线图，告警阈值 > 80% |
| 内存使用率 | 已用 / 总量 |
| 磁盘占用率 | 数据分区 / 日志分区 |
| 网络 I/O | 流入/流出速率 |
| 进程列表 | .NET 运行时进程状态 |
| API 请求统计 | TPS / 平均响应时间 / 错误率 |

### 3.19 仪表盘（Dashboard）

#### 操作路径

`首页 → Dashboard`

#### 功能说明

系统首页展示全局运行态势。

- 用户总数 / 今日活跃用户 / 在线用户数
- 文章总量 / 今日新增
- 待处理消息数
- 系统运行健康度评分
- 模块在线状态（Redis / Elasticsearch / MinIO）

## 四、内容管理模块（CMS）

### 4.1 站点管理

#### 操作路径

`内容管理 → 站点管理`

#### 功能说明

多站点能力: 一套 CMS 管理多个独立网站。

- 每个站点独立配置: 域名、模板、Logo、SEO 信息
- 站点数据天然隔离
- 站点切换: 顶部导航选择当前管理的站点

#### 站点配置字段

| 字段 | 说明 |
|------|------|
| 站点名称 | 前端展示 |
| 站点域名 | 用于前端路由/链接生成 |
| 默认站点 | 访客未指定站点时显示 |
| SEO 标题 | 页面 `<title>` 内容 |
| SEO 关键字 | `meta keywords` |
| SEO 描述 | `meta description` |
| 主题 | 选择不同的前端模板风格 |

### 4.2 栏目管理

#### 操作路径

`内容管理 → 栏目管理`

#### 功能说明

### 7.1 工作流概述

### 7.2 请假申请

#### 操作路径

`工作流 → 我的请假 → 新建请假`

#### 请假流程

```
员工提交申请
    │
    ▼
直属主管审批 ── 驳回 → 流程结束（员工可看到原因）
    │ 通过
    ▼
HR 备案 ── 驳回 → 退回员工
    │ 通过
    ▼
审批通过 → 通知申请人
```

#### 操作说明

1. 员工在"我的请假"中点击"新建"
2. 选择请假类型（事假/病假/年假/调休/婚假/产假/其他）
3. 填写开始时间、结束时间、请假事由
4. 提交后，状态变为"待审批"
5. 审批人在"待审批"中查看并操作（同意/驳回+原因）
6. 最终审批通过后，员工收到通知

- **审批操作**: 可查看申请详情，点击"同意"或"驳回（含原因）"
- 支持按申请人、时间范围、状态筛选

### 7.4 审批记录

查看任意申请的完整审批轨迹（谁在什么时间做了什么操作）。

## 十、高级功能

### 10.1 数据权限体系

数据权限是 JeeSite.NET 的核心特性之一。

**配置位置**: `角色管理 → 选中角色 → 数据权限`

| 数据权限类型 | 适用场景 | 实现原理 |
|------------|---------|---------|
| 全部数据 | 超级管理员 | 跳过所有过滤 |
| 本公司数据 | 子公司管理员 | 查询自动注入 `corp_code = 当前用户 corp_code` |
| 本公司及以下 | 集团型企业 | 递归查询子公司后注入 IN 条件 |
| 本机构数据 | 部门经理 | 注入 `org_code` 条件 |
| 本机构及以下 | 带机构层级的管理 | 递归查询子机构后注入 IN 条件 |
| 仅本人数据 | 普通员工 | 注入 `create_by = 当前用户 code` |
| 自定义 SQL | 特殊场景 | 手动编写 SQL 片段 |

### 10.2 字段权限

控制**同一个页面中**不同角色可以看到/编辑哪些字段。

**配置位置**: `角色管理 → 选中角色 → 字段权限`

典型应用: 财务数据中的"金额"字段对普通员工隐藏，但对财务人员可见。

### 10.3 短信/邮件验证码登录

除了常规的账号密码登录，JeeSite.NET 支持短信和邮件验证码登录。

#### 启用步骤

1. 在 `appsettings.json` 中配置:

```json
{
  "Sms": {
    "ApiUrl": "https://your-sms-provider.com/send",
    "ApiKey": "你的短信服务商 API Key"
  },
  "Email": {
    "SmtpHost": "smtp.example.com",
    "SmtpPort": 587,
    "Username": "noreply@example.com",
    "Password": "你的邮件密码"
  }
}
```

2. 在用户资料中为用户填写手机号或邮箱（用户管理 → 编辑用户）
3. 登录页下方点击"验证码登录"切换模式
4. 输入手机号/邮箱 → 点击"发送验证码" → 输入收到的 6 位验证码 → 登录

#### 安全性

- 同一手机号 1 分钟内最多发送 1 次，10 分钟内最多发送 5 次（限流防刷）
- 验证码 5 分钟有效，**一次使用即失效**（不可重复使用）
- 短信/邮件内容不包含用户密码信息

### 10.4 第三方登录（OAuth2）

#### 支持的平台

| 平台 | 配置方式 |
|------|---------|
| GitHub | ClientId + Secret |
| 微信 | AppId + AppSecret |
| 钉钉 | AppKey + AppSecret |

#### 登录流程

1. 点击登录页底部的第三方图标（GitHub / 微信 / 钉钉）
2. 浏览器跳转至服务商授权页
3. 用户授权后，通过 OAuth2 流程获取用户信息
4. 首次登录自动创建本地账号，下次登录直接绑定已有账号

### 10.5 CAS 单点登录

适用于企业内部已有 CAS 系统的场景。

- 访问 `/api/v1/sys/cas/login` → 跳转至 CAS 服务器
- CAS 认证成功后回调 `/api/v1/sys/cas/callback`
- 系统自动创建/登录本地用户

配置 `appsettings.json`:

```json
{
  "Cas": {
    "ServerUrl": "https://cas.example.com/cas",
    "ServiceUrl": "https://your-app.com/api/v1/sys/cas/callback"
  }
}
```

### 10.6 LDAP 认证

适用于企业内部 Active Directory / OpenLDAP。

配置 `appsettings.json`:

```json
{
  "Ldap": {
    "Server": "ldap.example.com",
    "Port": 389,
    "BaseDn": "OU=Users,DC=example,DC=com",
    "Domain": "EXAMPLE"
  }
}
```

### 10.7 MCP Server（JSON-RPC 2.0）

MCP（Model Context Protocol）允许 AI 客户端通过标准化协议调用 JeeSite.NET 的业务能力。

- **端点**: `POST /api/v1/sys/mcp/jsonrpc`
- **协议**: JSON-RPC 2.0
- **认证方式**: JWT Bearer Token（需具备 `sys:mcp:invoke` 权限）

供高级开发/集成场景使用。

### 10.8 全文搜索（Elasticsearch）

- 文章内容自动索引到 Elasticsearch
- 支持中文分词（IK Analyzer）
- CMS 前端搜索框 / `POST /api/v1/sys/search` API 均可调用
- 支持按时间范围、栏目、关键字混合检索

### 11.7 API 鉴权机制

**登录流程**:

```
客户端 POST /api/v1/sys/auth/login (loginCode + password)
        │
        ▼
    服务端校验
        │通过
        ▼
    生成 JWT Token（含用户ID/名称/权限）→ 返回前端
        │
        ▼
    前端存储 Token（localStorage）
        │
        ▼
    后续请求 Header: Authorization: Bearer <token>
```

**Token 有效期**:

- 默认 12 小时
- 服务端可通过"在线用户 → 踢出"强制失效
- 主动退出: 调用 `POST /api/v1/sys/auth/logout`

**新增 API 的鉴权装饰**:

```csharp
[Authorize]              // 需登录
[Permission("biz:xxx:view")] // 需特定权限
[HttpGet("list")]
public async Task<ApiResult<...>> List(...)
```

## 十二、常见问题 FAQ

**Q1. 忘记 admin 密码怎么办？**

> 答: 有三种方式：
> 1. 使用"忘记密码"页面（需配置邮件服务）
> 2. 使用管理员账号的短信/邮件验证码登录
> 3. 直接在数据库执行 `UPDATE sys_user SET password = MD5('新密码') WHERE login_code = 'admin'`

**Q2. 新增的菜单/功能为什么用户看不到？**

> 答: 常见原因有三种:
> - 菜单的"显示/隐藏"属性设置错误（应为显示）
> - 用户的角色未包含该菜单权限
> - 用户登录后未刷新缓存，退出重新登录即可

**Q3. 如何限制某个角色只能看自己创建的数据？**

> 答: 在角色管理 → 数据权限 → 选择"仅本人数据"。系统在查询时会自动注入 `create_by = 当前用户` 的过滤条件。

**Q4. 富文本编辑器的图片粘贴为什么上传失败？**

> 答: 可能原因:
> - 文件超过 `upload.maxSize` 限制（默认 10MB）
> - 文件类型不在白名单内
> - 存储服务（MinIO/本地）未正确配置
> - 检查 `EditorController` 后端日志

**Q5. 如何在多台服务器上部署 JeeSite.NET？**

> 答: 关键要点:
> 1. 使用 **同一个** Redis 实例（保证 JWT 吊销、Session 共享、缓存一致性）
> 2. 使用 **同一个** 数据库
> 3. 文件上传推荐使用 MinIO/OSS 等共享存储，而非本地磁盘
> 4. 通过 Nginx 或 Azure Front Door 等做负载均衡
> 5. SignalR 需配置 Backplane（Redis StackExchange.Redis）

**Q6. Docker Compose 启动后首次访问数据为空？**

> 答: 首次启动后数据库是空的。系统包含 Seed Data 初始化逻辑，首次启动会自动插入:
> - 默认管理员 admin/admin
> - 默认菜单树
> - 默认字典数据
> - 默认机构/公司/岗位
> - 默认站点与栏目
>
> 请等待容器启动完成后（约 1-2 分钟）再访问。

**Q7. 短信发送失败如何排查？**

> 答: 排查路径:
> 1. 确认 `appsettings.json` 中 `Sms:ApiUrl` / `Sms:ApiKey` 已正确配置
> 2. 检查 `SmsController` / `SmsService` 的日志
> 3. 用 Postman/curl 直接调用短信服务商 API 验证配置
> 4. 确认接收方手机号未被服务商拉黑
> 5. 查看是否被限流（1 分钟 1 次 / 10 分钟 5 次）

**Q8. 为什么修改了数据库中的 sys_menu 数据，前端没有更新？**

> 答: 菜单使用 FusionCache 两级缓存。处理方式:
> - 进入"系统管理 → 缓存管理"，清理菜单缓存
> - 或直接重启后端容器/进程
> - 前端需刷新页面重新拉取

**Q9. 生产环境部署需要注意什么安全配置？**

> 答: 详见[部署文档](file:///d:/Projects/jeesite.net/docs/07-部署运维/部署文档.md)，关键要点:
> - 立即修改 admin 默认密码
> - 修改 JWT Secret（必须修改！）
> - 启用 HTTPS，配置 HSTS
> - 设置 nginx/负载均衡的 WAF（Web 应用防火墙）
> - 数据库账号使用最小权限原则
> - 定期备份（日备+周备+月备）
> - 监控告警接入（CPU/内存/磁盘/错误率）

**Q10. 我的数据可以从 JeeSite5（Java 版）迁移过来吗？**

> 答: 可以。JeeSite.NET 的表结构设计与 JeeSite5 保持一致，核心表（sys_user / sys_role / sys_menu / sys_dict / sys_office / sys_company 等）字段完全对齐。
> 迁移步骤:
> 1. 通过 SQL Server 的导入导出功能迁移数据
> 2. 字段名称需适配（JeeSite5 部分表字段命名可能不同）
> 3. 密码字段需特殊处理（JeeSite5 使用自有算法，需用户重新设置密码）

## 十三、故障排查

### 13.1 后端无法启动

| 现象 | 排查方向 |
|------|---------|
| 数据库连接失败 | 检查 `ConnectionStrings:DefaultConnection` 是否正确；SQL Server 服务是否运行；SQL 身份认证是否开启 |
| Redis 连接失败 | `FusionCache:Redis` 配置；Redis 服务是否运行；端口 6379 是否开放 |
| 端口被占用 | 5000 端口可能被其他进程占用；`netstat -ano | findstr :5000` |
| 未授权异常 | JWT Secret 为空；检查 `Jwt:Secret` 配置 |

### 13.2 前端无法访问后端

| 现象 | 排查方向 |
|------|---------|
| 401 Unauthorized | Token 未在请求头中携带；Token 已过期；`VITE_API_BASE` 指向的地址不正确 |
| CORS 错误 | 后端未正确配置跨域；前端请求的 Origin 不在白名单 |
| 500 Internal Server Error | 后端代码抛出未处理异常；查看 Swagger 或后端日志 |
| SignalR 连接不上 | WebSocket 被代理/Nginx 拦截；需配置 `Upgrade: websocket` |

### 13.3 文件上传失败

| 现象 | 排查方向 |
|------|---------|
| 文件大小超过限制 | 调整 `upload.maxSize`；同时检查 nginx 的 `client_max_body_size` |
| 图片上传后无法预览 | 检查存储服务（MinIO/本地）的文件权限与 URL 是否可公开访问 |
| 文件扩展名被禁止 | 在 `FileSecurityUtil` 的白名单中添加你需要的类型 |

### 13.4 登录后无菜单显示

通常是**菜单权限**未分配:

1. 以 admin 登录
2. 进入"系统管理 → 角色管理"
3. 编辑对应角色
4. 在"菜单权限"中勾选需要展示的菜单
5. 被影响的用户重新登录

### 13.5 性能问题排查

| 问题 | 排查工具 | 常见原因 |
|------|---------|---------|
| 页面加载缓慢 | 浏览器 F12 → Network 面板 | 前端资源未压缩；API 响应慢；字体/大图片 |
| API 响应 > 2s | Swagger / Postman / 后端日志 | 缺少数据库索引；N+1 查询；未走缓存 |
| 内存持续增长 | dotnet-dump / 任务管理器 | 缓存未设置过期；DbContext 生命周期过长；大对象未释放 |
| CPU 高 | top / 任务管理器 | 死循环；密集计算；GC 频繁；检查慢日志 |
