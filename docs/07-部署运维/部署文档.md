# JeeSite.NET 部署运维指南

## 环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| .NET SDK | 10.0+ | 后端运行时 |
| Node.js | 20 LTS+ | 前端构建 |
| pnpm | 9+ | 前端包管理 |
| SQL Server | 2022+ | 主数据库 |
| Redis | 7+ | 缓存 (L2 + Session) |
| Docker | 24+ | 容器化部署 (可选) |

## 开发环境搭建

### 1. 后端

```bash
# 还原依赖
dotnet restore

# 编译
dotnet build

# 运行 API 服务
dotnet run --project src/JeeSiteNET.Web.Api

# API 地址: http://localhost:5000
# Swagger:   http://localhost:5000/swagger
```

### 2. 前端

```bash
cd frontend

# 安装依赖
pnpm install

# 开发模式运行
pnpm dev

# 前端地址: http://localhost:5173
```

### 3. 数据库初始化

首次运行会自动创建数据库并初始化种子数据。如需手动执行迁移：

```bash
dotnet ef database update --project src/JeeSiteNET.Infrastructure --startup-project src/JeeSiteNET.Web.Api
```

创建新迁移：

```bash
dotnet ef migrations add {MigrationName} --project src/JeeSiteNET.Infrastructure --startup-project src/JeeSiteNET.Web.Api
```

### 4. Redis（可选）

本地开发默认使用内存缓存。如需 Redis：

```bash
docker run -d --name redis -p 6379:6379 redis:7-alpine
```

## Docker Compose 部署（一键启动）

```bash
# 构建并启动全部服务
docker compose up -d

# 查看日志
docker compose logs -f

# 停止服务
docker compose down

# 重建单服务
docker compose build webapi
docker compose up -d webapi
```

### 服务端口

| 服务 | 端口 | 说明 |
|------|------|------|
| SQL Server | 1433 | 数据库 |
| Redis | 6379 | 缓存 |
| WebApi | 5000 | 后端 API |
| Frontend | 3000 | 前端 Nginx |

### Docker 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `ConnectionStrings__DefaultConnection` | SQL Server 连接串 | 数据库连接 |
| `Redis__Connection` | `redis:6379` | Redis 地址 |
| `Redis__InstanceName` | `JeeSiteNET` | Redis 实例前缀 |
| `Jwt__Secret` | 32位密钥 | JWT 签名密钥 |

## 生产部署

### 后端独立部署

```bash
# 发布
dotnet publish src/JeeSiteNET.Web.Api -c Release -o publish

# 运行
ASPNETCORE_ENVIRONMENT=Production \
ASPNETCORE_URLS=http://*:5000 \
ConnectionStrings__DefaultConnection="..." \
Redis__Connection="..." \
Jwt__Secret="..." \
dotnet publish/JeeSiteNET.Web.Api.dll
```

### 前端独立部署

```bash
cd frontend

# 构建
pnpm build

# 输出目录: frontend/dist/
# 配置 Nginx 指向该目录并代理 /api/ 到后端
```

Nginx 配置示例：

```nginx
server {
    listen 80;
    server_name example.com;

    root /var/www/jeesite/dist;
    index index.html;

    # 前端静态文件
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://localhost:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Swagger
    location /swagger/ {
        proxy_pass http://localhost:5000;
    }
}
```

### HTTPS 配置

推荐使用反向代理（如 Nginx）终结 SSL：

```nginx
server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/ssl/certs/example.com.pem;
    ssl_certificate_key /etc/ssl/private/example.com.key;

    # 其余同 HTTP 配置
}
```

## 日志与监控

### 日志文件

- 后端日志输出到 `stdout`（Docker 环境）或 `logs/` 目录
- 请求日志记录在 `Sys_Log` 表（`LogType = 'access'`）

### 健康检查

API 健康检查端点（Docker Compose 已配置）：

```
GET /healthz → 200 OK
```

## 常见问题

### 1. SQL Server 连接失败

确认 SQL Server 已启动且 TCP/IP 协议已启用：

```sql
-- 检查端口监听
EXEC sys.sp_readerrorlog 0, 1, 'listening';
```

### 2. 跨域问题

开发环境允许 `http://localhost:5173`。生产环境需修改 `Program.cs` 中的 CORS 配置。

### 3. 数据库迁移冲突

```bash
# 回滚最近迁移
dotnet ef migrations remove --project src/JeeSiteNET.Infrastructure --startup-project src/JeeSiteNET.Web.Api

# 重新创建
dotnet ef migrations add Initial --project src/JeeSiteNET.Infrastructure --startup-project src/JeeSiteNET.Web.Api
```

### 4. Seed Data 未初始化

首次启动自动初始化。如需要手动触发：

```bash
curl -X POST http://localhost:5000/api/v1/sys/init/seed
```
