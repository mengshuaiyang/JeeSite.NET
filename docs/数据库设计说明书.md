# JeeSite.NET 数据库设计说明书

> 版本: 1.4 | 更新日期: 2026-06-07 | 数据库: SQL Server

> 更新: P1-2 公司管理 + 行政区划，新增 3 张表 Sys_Company / Sys_Company_Office / Sys_Area，Sys 模块增至 24 张表

---

## 一、概述

本文档描述 JeeSite.NET 平台的数据库结构。系统共包含 **33 张表**，分布在 **5 个模块**中。

### 1.1 实体继承体系

所有实体继承自三层抽象基类，形成统一的审计和时间戳字段标准：

```
BaseEntity                        → CreateBy, CreateDate, UpdateBy, UpdateDate, Remarks
  └── DataEntity                  → + Status (varchar(1), default '0')
        └── TreeEntity            → + ParentCode, ParentCodes, TreeSort, TreeSorts,
                                     TreeLeaf, TreeLevel, TreeNames (7 个树形字段)
```

| 基类 | 使用该基类的表 |
|------|-------------|
| **BaseEntity** | Sys_Log, CodeGen_Table, CodeGen_TableColumn, Tasks_JobLog |
| **DataEntity** | Sys_User, Sys_Role, Sys_Menu, Sys_Organization, Sys_Tenant, Sys_Post, Sys_Module, Sys_Dict_Type, Sys_Dict_Data, Sys_Config, Cms_Site, Cms_Category, Cms_Article, Bpm_WorkflowForm, Bpm_ApprovalRecord, Tasks_Job |
| **TreeEntity** | Sys_Menu, Sys_Organization, Cms_Category |
| *(无基类)* | Sys_User_Role, Sys_Role_Menu |

### 1.2 接口继承体系

实体通过接口实现通用字段分组，按需组合：

```
ICorpEntity   → CorpCode, CorpName                          (多租户公司标识)
IExtendEntity → ExtendS1~S8, ExtendI1~I4, ExtendF1~F4,     (扩展字段，21 字段)
                ExtendD1~D4, ExtendJson
```

| 接口 | 实现该接口的表 |
|------|-------------|
| **ICorpEntity** | Sys_User, Sys_Role, Sys_Menu, Sys_Organization, Sys_Post, Sys_Log, Sys_Dict_Data, Sys_Role_Data_Scope, Sys_User_Data_Scope, Sys_Employee, Sys_Company, Cms_Site, Cms_Category, Cms_Article |
| **IExtendEntity** | Sys_User, Sys_Role, Sys_Menu, Sys_Organization, Sys_Dict_Data, Sys_Employee, Sys_Company, Cms_Category |

### 1.3 全局公共字段

所有继承 BaseEntity 的表均包含以下列：

| 列名 | 类型 | 可为空 | 说明 |
|------|------|--------|------|
| `CreateBy` | varchar(100) | 是 | 创建者编码 |
| `CreateDate` | datetime | 是 | 创建时间 |
| `UpdateBy` | varchar(100) | 是 | 更新者编码 |
| `UpdateDate` | datetime | 是 | 更新时间 |
| `Remarks` | varchar(500) | 是 | 备注说明 |

DataEntity 额外增加：

| 列名 | 类型 | 可为空 | 默认值 | 说明 |
|------|------|--------|--------|------|
| `Status` | varchar(1) | 是 | `'0'` | 状态（0=正常, 1=停用） |

TreeEntity 额外增加：

| 列名 | 类型 | 可为空 | 默认值 | 说明 |
|------|------|--------|--------|------|
| `ParentCode` | varchar(100) | 否 | `'0'` | 父级编码 |
| `ParentCodes` | varchar(2000) | 否 | `''` | 所有父级编码链 |
| `TreeSort` | decimal(18,2) | 否 | `1000` | 排序号 |
| `TreeSorts` | varchar(2000) | 否 | `''` | 排序号链 |
| `TreeLeaf` | varchar(1) | 否 | `'1'` | 是否叶子节点 |
| `TreeLevel` | decimal(18,2) | 否 | `0` | 树层级 |

ICorpEntity 接口字段（多租户标识）：

| 列名 | 类型 | 可为空 | 默认值 | 说明 |
|------|------|--------|--------|------|
| `CorpCode` | varchar(100) | 是 | | 公司编码 |
| `CorpName` | varchar(200) | 是 | | 公司名称 |

IExtendEntity 接口字段（扩展字段，用于自定义业务属性）：

| 列名 | 类型 | 可为空 | 说明 |
|------|------|--------|------|

| `ExtendF1` ~ `ExtendF4` | decimal(18,4) | 是 | 扩展浮点数 1~4 |
| `ExtendD1` ~ `ExtendD4` | datetime | 是 | 扩展日期 1~4 |
| `ExtendJson` | nvarchar(max) | 是 | 扩展 JSON |

## 十一、数据权限体系

JeeSite.NET 完整复刻 JeeSite 5 的数据权限控制能力，支持以下 7 种范围（`DataScopeType`）：

| 枚举值 | 中文 | SQL 行为 |
|--------|------|----------|
| `All` | 全部数据 | 不过滤 |
| `Company` | 本公司 | `CorpCode = @当前租户` |
| `CompanyAndChild` | 本公司及下属 | `CorpCode IN (本公司 + 下属公司树)` |
| `Office` | 本部门 | `OfficeCode = @当前部门` |
| `OfficeAndChild` | 本部门及下属 | `OfficeCode IN (本部门 + 下级部门树)` |
| `Self` | 仅本人 | `CreateBy = @当前用户` |
| `Custom` | 自定义 | `RuleConfig` 中保存的 SQL 片段 |

### 11.1 优先级

数据权限的解析按 **用户级 → 菜单级 → 角色级** 顺序优先级匹配：

```
1. Sys_User_Data_Scope（用户级，最高优先级）
         ↓ 未配置
2. Sys_Menu_Data_Scope（菜单级，按当前菜单匹配）
         ↓ 未配置
3. Sys_Role.DataScope（角色级，全局兜底）
         ↓ 未配置
4. 默认 All（全部数据）
```

### 11.2 API 端点

| Method | URL | 说明 |
|--------|-----|------|
| GET | `/api/v1/sys/role-data-scope/role/{roleCode}` | 查询角色的数据权限配置列表 |

### 11.3 服务层使用

```csharp
// 在 Service 中应用数据权限
public async Task<PageResult<UserDto>> PageAsync(PageRequest req)
{
    var query = _db.Set<User>().AsNoTracking();
    query = (IQueryable<User>)_dataScopeService.ApplyDataScope(query, "user");
    return await query.Select(...).ToPageResultAsync(req);
}
```
