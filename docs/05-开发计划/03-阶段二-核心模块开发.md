# 阶段二: 核心模块开发

> 时间: 第5-10周
> 目标: 完成 Sys 系统管理模块，实现完整权限体系

## 1. 第5-6周: 组织与用户管理

| 任务 | 领域实体 | 服务 | 功能点 |
|---|---|---|---|
| 机构管理 | Organization (树) | OrganizationService | CRUD + 树结构 + 级联维护 |
| 岗位管理 | Post | PostService | CRUD + 排序 |
| 用户管理 | User (多用户类型) | UserService | CRUD + 密码 + 头像 + 用户类型 |
| 用户附属机构 | UserOrg | UserOrgService | 多部门关联 |

**关键实现**:
- 树表辅助字段自动级联更新 (TreeEntityInterceptor)
- 数据权限控制 (管理的权限)
- 防越级授权检查

## 2. 第7-8周: 角色与权限管理

| 任务 | 领域实体 | 服务 | 功能点 |
|---|---|---|---|
| 角色管理 | Role | RoleService | CRUD + 用户类型 |
| 角色授权 | RoleMenu | RoleMenuService | 菜单 + 权限树分配 |
| 用户角色分配 | UserRole | UserRoleService | 批量分配 |
| 菜单管理 | Menu (树) | MenuService | CRUD + 树 + 权重 |
| 数据权限规则 | DataPermission | DataPermissionService | 规则配置 + 范围类型 |

**关键实现**:
- RBAC 权限模型完整实现
- 角色权限缓存 + 变更清理
- 二级管理员权限范围控制
- 权限双重验证 (前端菜单隐藏 + 后端 API 拦截)

## 3. 第9周: 系统设置

| 任务 | 实体 | 功能点 |
|---|---|---|
| 字典管理 | DictType / DictData | CRUD + 缓存 |
| 参数配置 | Config | CRUD + 配置覆盖 |
| 行政区划 | Area (树) | 树表维护 |
| 模块管理 | Module | 模块注册/启停 |
| 国际化管理 | I18n | 多语言维护 |

## 4. 第10周: 系统监控与安全

| 任务 | 实体 | 功能点 |
|---|---|---|
| 操作日志 | Log | 自动记录 + 前后数据diff |
| 在线用户 | 运行时数据 | 在线查询 + 强制踢出 |
| 密码策略 | 配置驱动 | 5级密码强度验证 |
| 安全审计 | 审计查询 | 账号/菜单/用户审计 |
| 缓存监控 | 运行时 | 缓存查询/清理 |
