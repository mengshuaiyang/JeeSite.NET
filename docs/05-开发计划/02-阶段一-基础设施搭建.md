# 阶段一: 基础设施搭建

> 时间: 第1-4周
> 目标: 完成项目骨架、核心层、基础设施层，可编译运行

## 1. 第1周: 项目初始化

| 任务 | 产出 |
|---|---|
| 创建解决方案和项目结构 | JeeSite.NET.sln + 所有空项目 |
| 配置 Directory.Build.props | 统一编译属性 |
| 配置 NuGet 包引用 | 核心依赖清单 |
| 创建 .gitignore / .editorconfig | 代码规范配置 |
| 配置 CI/CD (GitHub Actions) | 自动编译 |

## 2. 第2周: 核心层 (Core)

| 任务 | 产出 |
|---|---|
| `ApiResult<T>` 统一响应 | 基础响应封装 |
| `PageRequest<T>` / `PageResult<T>` | 分页基类 |
| `BaseEntity` / `DataEntity` | 实体基类 |
| `TreeEntity` + 辅助字段 | 树表基类 |
| `IModuleDescriptor` / `IModuleInstaller` | 模块化接口 |
| `ModuleLoader` | 模块加载器 |
| `ICurrentUser` / `IDataScope` | 安全抽象 |
| 通用工具类 (IdGenerator / StringUtils) | 工具集 |

## 3. 第3周: 基础设施层 (Infrastructure)

| 任务 | 产出 |
|---|---|
| `JeeSiteDbContext` + 审计拦截器 | 核心 DbContext |
| TreeEntityInterceptor | 树辅助字段自动维护 |
| 多租户 Global Query Filter | 租户隔离 |
| 软删除拦截器 | 逻辑删除过滤 |
| FusionCache 集成 | L1+L2 缓存 |
| JWT 认证 + Identity 集成 | 认证授权 |
| Serilog 配置 | 结构化日志 |

## 4. 第4周: Web 骨架

| 任务 | 产出 |
|---|---|
| Web API 项目结构 | Controllers/Filters/Middlewares |
| Web Host 项目结构 | MVC + Razor Pages |
| 全局异常过滤器 | 错误处理 |
| 请求日志中间件 | 访问记录 |
| Swagger/OpenAPI 配置 | API 文档 |
| 数据库迁移初始化 | 初始 Migration |
| Docker Compose 配置 | Redis/RabbitMQ/MinIO |

## 5. 交付检查清单

- [ ] 解决方案可编译无错误
- [ ] `dotnet build` 通过
- [ ] Docker Compose 依赖启动正常
- [ ] Web API 启动并返回健康检查
- [ ] Swagger 界面可访问
- [ ] 数据库迁移自动执行
- [ ] 单元测试框架可用
