## 三、未实现功能清单（按优先级分组）

### P0 - 核心架构缺失（必须优先补齐）

#### 1. Extend 扩展字段体系

**描述：** JeeSite 5 中，`Company`、`Office`、`Role`、`Menu`、`User`、`DictData`、`CmsArticleData`、`CmsCategory` 等主要实体都包含扩展字段：8 个字符串、4 个整数、4 个小数、4 个日期、1 个 JSON。这允许业务自定义扩展而无需改表结构。

**影响范围：** Sys_Organization, Sys_Role, Sys_Menu, Sys_User, Sys_Dict_Data, Cms_Category + 所有新建实体

**工作量：** 中（约 2-3 天）
- 设计 `ExtendEntity` 基类（ef1 ~ ef8, ei1 ~ ei4, ef1 ~ ef4, ed1 ~ ed4, ee_json）
- 修改现有实体继承链 6 处
- 更新 EntityConfiguration 添加映射
- 更新前端组件

#### 2. Corp 多租户公司标识

**描述：** 除字典、配置、日志等少数表外，几乎所有 JeeSite 5 表都有 `corp_code` 和 `corp_name` 字段，用于标识数据所属的公司（租户）。

**影响范围：** 所有 DataEntity 表

**工作量：** 中（约 1-2 天）
- 在 DataEntity 基类中添加 `CorpCode`、`CorpName`
- 创建数据过滤拦截器
- 更新 EntityConfiguration

#### 3. 数据权限体系

**描述：** JeeSite 5 有完整的数据权限控制，包括：
- `Sys_Role_Data_Scope` — 角色级数据规则（按公司、部门、自定义）
- `Sys_User_Data_Scope` — 用户级数据规则
- `Sys_Menu_Data_Scope` — 菜单级数据规则
- 5 种范围类型：ALL、COMPANY_AND_CHILD、COMPANY、OFFICE_AND_CHILD、SELF、CUSTOM

**工作量：** 大（约 4-5 天）
- 新建 3 张表和相关实体
- 实现 `DataScopeService` 完整逻辑
- 在查询中嵌入数据权限过滤
- 前端数据权限配置页面

#### 4. 字段权限体系

**描述：** `Sys_Role_Field_Scope` 允许按角色控制哪些用户能看到哪些字段。例如，普通员工看不到薪资字段。

**工作量：** 中（约 2-3 天）
- 新建 `Sys_Role_Field_Scope` 表
- 实现字段级过滤逻辑
- 前端字段权限配置界面

### P1 - 核心业务功能（关键业务缺失）

#### 5. 员工管理

**描述：** JeeSite 5 有完整的员工管理子模块：
- `Sys_Employee` — 员工信息（工号、姓名、英文名、归属机构、公司）
- `Sys_Employee_Office` — 员工附属机构（一人多部门）
- `Sys_Employee_Post` — 员工岗位（一人多岗）
- `Sys_EmpUser` — 员工账号关联（一个员工可以关联多个系统账号）
- `Sys_Post_Role` — 岗位角色关联（通过岗位继承角色）

**表数量：** 5 张

**工作量：** 大（约 5-7 天）
- 新建 5 张实体表及配置
- 实现完整的 EmployeeService
- 实现 EmpUser 组合视图
- 实现岗位角色继承
- 前端员工管理、岗位分配页面

#### 6. 公司管理

**描述：** `Sys_Company` 是树形结构（TreeEntity），用于管理公司/单位层级，并关联到机构（Office）。一个公司可以有多个部门。包含 Extend 字段。

**表数量：** 2 张（Sys_Company, Sys_Company_Office）

**工作量：** 中（约 2-3 天）
- 新建 Company 实体（TreeEntity）
- 新建 CompanyOffice 关联实体
- 实现 CompanyService
- 前端公司管理树页面

#### 7. 行政区划管理

**描述：** `Sys_Area` 是树形结构，预装载中国省/市/区/街道数据（约 4000 条）。支持按名称、编码查询和模糊搜索。

**表数量：** 1 张 + 预置数据

**工作量：** 中（约 2-3 天）
- 新建 Area 实体（TreeEntity）
- 准备中国行政区划数据种子
- 实现 AreaService
- 前端区域选择组件

## 六、注意事项

### 架构差异要点

| 方面 | JeeSite 5 (Java) | JeeSite.NET 当前 | 建议 |
|------|-----------------|------------------|------|
| ORM | MyBatis + 自定义查询 | EF Core Fluent API | 保留 EF Core |
| 多租户 | CorpCode 字段过滤 | TenantContext + TenantCode | 补充 CorpCode 支持 |
| 扩展字段 | Extend 值对象内嵌 | 无 | 添加 ExtendEntity 支持 |
| 数据权限 | SQL 级动态拼接 | IDataScope 接口 | 完善 DataScopeService |
| 缓存 | EhCache/Redis | FusionCache (L1+L2) | 继续使用 FusionCache |
| 代码生成 | Beetl 模板引擎 | Razor 模板 | 继续使用 Razor |
| 前端 | Vue 3 + Element Plus | 未开发 | 建议使用 Vue 3 + Ant Design Vue |
| 工作流 | Activiti/Flowable | Elsa Workflow | 继续使用 Elsa |

### 迁移建议

1. **不要直接照搬** Java 代码。JeeSite 5 经过多年迭代，包含一些历史遗留设计（如 MyBatis 的 XML 配置）。我们应当取其设计理念，用 .NET 最佳实践实现。

2. **优先补齐 P0**。Extend 和 DataScope 贯穿所有模块，越早补齐后续模块开发的成本越低。

3. **前端同步开发**。建议新功能同时设计 API 和前端组件，避免 API 完成后前端积压。

4. **分模块迭代**。每个功能模块应在 1-2 周内可交付，避免长时间不发布。
