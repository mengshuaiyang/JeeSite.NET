# 代码规范与命名约定

> 继承 JeeSite 5 的开发规范，结合 .NET 社区最佳实践和 Microsoft 官方 C# 编码规范。

## 1. C# 编码规范

### 1.1 命名规则

| 类别 | 规则 | 示例 |
|---|---|---|
| 命名空间 | PascalCase + 点分 | `JeeSiteNET.Modules.Sys` |
| 类/结构体 | PascalCase | `UserService`, `PageRequest` |
| 接口 | `I` + PascalCase | `IUserRepository`, `ICrudService` |
| 方法 | PascalCase | `GetAsync()`, `FindListAsync()` |
| 属性 | PascalCase | `UserName`, `CreateDate` |
| 公共字段 | PascalCase | `MaxRetryCount` |
| 私有字段 | `_` + camelCase | `_userRepository`, `_cache` |
| 局部变量 | camelCase | `userList`, `pageResult` |
| 常量 | PascalCase | `DefaultPageSize` |
| 参数 | camelCase | `request`, `entity` |
| 枚举 | PascalCase | `UserType`, `LogLevel` |
| 枚举值 | PascalCase | `Employee`, `Admin` |
| 异步方法 | `Async` 后缀 | `SaveAsync()`, `DeleteAsync()` |

### 1.2 文件结构规范

```csharp
// 文件范围内命名空间 (File-scoped namespace)
namespace JeeSiteNET.Modules.Sys.Application;

/// <summary>
/// 用户应用服务
/// </summary>
public class UserService : CrudService<User, UserDto>, IUserService
{
    private readonly IUserRepository _userRepository;
    private readonly ICacheService _cache;

    // 构造函数注入
    public UserService(IUserRepository userRepository, ICacheService cache)
    {
        _userRepository = userRepository;
        _cache = cache;
    }

    // 公共方法
    public async Task<UserDto> GetByLoginCodeAsync(string loginCode)
    {
        var user = await _userRepository.GetByLoginCodeAsync(loginCode);
        return MapToDto(user);
    }

    // 私有方法
    private static UserDto MapToDto(User user)
    {
        return new UserDto
        {
            UserCode = user.UserCode,
            UserName = user.UserName,
        };
    }
}
```

### 1.3 代码格式规则

- 缩进: 4个空格 (不用 Tab)
- 花括号: 另起一行 (Allman 风格)
- 行长度: 建议不超过 120 字符
- 文件编码: UTF-8 (含 BOM)
- using 语句: 文件顶部，按 `System → NuGet → 内部` 分组

### 1.4 Async 命名规范

```csharp
// ✅ 正确
public Task<User> GetAsync(string id);
public Task<List<User>> FindListAsync(User entity);
public Task SaveAsync(User entity);
public Task DeleteAsync(User entity);

// ❌ 错误 (同步方法名)
public Task<User> GetUser(string id);
```

## 2. TypeScript 编码规范

### 2.1 命名规则

| 类别 | 规则 | 示例 |
|---|---|---|
| 接口 | `I` + PascalCase | `IUser`, `IPageRequest` |
| 类型别名 | PascalCase + `Type` | `UserType`, `ListQueryType` |
| 类/组件 | PascalCase | `UserService`, `UserTable` |
| 函数 | camelCase | `fetchList()`, `handleSubmit()` |
| 变量 | camelCase | `userList`, `currentPage` |
| 常量 | UPPER_SNAKE_CASE | `DEFAULT_PAGE_SIZE` |
| 枚举 | PascalCase | `UserType` |
| 枚举值 | PascalCase | `Employee`, `Admin` |
| 文件(组件) | PascalCase | `UserTable.vue` |
| 文件(脚本) | camelCase | `userService.ts` |

## 3. SQL 开发规范

- 关键字大写: `SELECT`, `INSERT`, `UPDATE`, `DELETE`, `FROM`, `WHERE`
- 表名/列名小写: `sys_user`, `user_code`
- 缩进对齐: 子句之间换行对齐
- 表别名: 表名首字母缩写

```sql
SELECT a.user_code,
       a.user_name,
       a.login_code,
       r.role_name
FROM   sys_user a
       LEFT JOIN sys_user_role ur ON ur.user_code = a.user_code
       LEFT JOIN sys_role r ON r.role_code = ur.role_code
WHERE  a.status = '0'
  AND  a.user_name LIKE '%张%'
ORDER BY a.create_date DESC;
```
