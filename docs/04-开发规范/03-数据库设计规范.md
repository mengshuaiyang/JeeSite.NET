# 数据库设计规范

> 继承 JeeSite 5 的数据库设计规范，结合 EF Core 技术特性。

## 1. 命名规范

### 1.1 表命名

- 格式: `{模块前缀}_{表名}`
- 模块前缀: Sys_ / Bpm_ / Cms_ / File_
- 词汇间用下划线分隔
- 使用英文单词或常用缩写
- 示例: `Sys_User`, `Sys_Role_Menu`, `Bpm_Process_Instance`

### 1.2 列命名

- 小写 + 下划线: `user_code`, `user_name`, `create_date`
- 主键: `{表名缩写}_code` (如 `user_code`)
- 外键: 引用表主键名
- 布尔类型: `is_xxx` / `has_xxx` (如 `is_sys`, `is_show`)

### 1.3 索引命名

- 主键索引: `PK_{表名}`
- 唯一索引: `UK_{表名}_{列名}`
- 普通索引: `IX_{表名}_{列名}`

## 2. 设计规范

### 2.1 每表必备字段

| 字段名 | 类型 | 说明 |
|---|---|---|
| `status` | CHAR(1) | 状态: 0正常 1删除 2停用 |
| `create_by` | VARCHAR(100) | 创建者编码 |
| `create_date` | DATETIME | 创建时间 |
| `update_by` | VARCHAR(100) | 更新者编码 |
| `update_date` | DATETIME | 更新时间 |
| `remarks` | VARCHAR(500) | 备注 (可选) |

### 2.2 主键设计

- 优先使用有意义的编码 (非 UUID)
- 编码格式: 数字/字母组合，便于运维识别
- 组织机构: `370000` (层级编码)
- 字典: `dict_user_type_employee`
- EF Core 配置示例:

```csharp
builder.HasKey(e => e.UserCode);
builder.Property(e => e.UserCode)
    .HasMaxLength(100)
    .ValueGeneratedNever();  // 手动赋值
```

### 2.3 字段类型规范

| C# 类型 | 数据库类型 | 说明 |
|---|---|---|
| string | NVARCHAR(n) | 字符串 |
| string (定长) | CHAR(n) | status, is_show |
| int | INT | 整数 |
| long | BIGINT | 长整数 |
| decimal | DECIMAL(18,2) | 金额 |
| decimal | DECIMAL(10) | 排序号 |
| DateTime | DATETIME2 | 时间 |
| bool | BIT | 布尔 |
| byte[] | VARBINARY(MAX) | 二进制 |

### 2.4 字段长度规范

| 内容 | 长度 | 示例 |
|---|---|---|
| 编码/主键 | VARCHAR(100) | user_code |
| 名称 | VARCHAR(200) | user_name, role_name |
| 简短输入 | VARCHAR(100) | phone, status |
| 邮箱 | VARCHAR(300) | email |
| URL | VARCHAR(500) | avatar, menu_href |
| 较长的文本 | VARCHAR(2000) | parent_codes |
| 大文本 | TEXT | params, diff_data |

## 3. 树表规范

参考 `02-架构设计/04-数据库架构设计.md` 中的树表设计。

### 3.1 树表必备字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `parent_code` | VARCHAR(100) | 上级编码 |
| `parent_codes` | VARCHAR(2000) | 所有上级编码链 |
| `tree_sort` | DECIMAL(10) | 本级排序 |
| `tree_sorts` | VARCHAR(2000) | 完整排序号 |
| `tree_leaf` | CHAR(1) | 是否叶子节点 |
| `tree_level` | DECIMAL(4) | 层次级别 |
| `tree_names` | VARCHAR(2000) | 全路径名称 |

## 4. 使用规范

- 不使用存储过程和自定义函数 (业务逻辑在 C# 层处理)
- 不使用数据库触发器 (使用 EF Core 拦截器)
- 查询条件参数化，杜绝 SQL 注入
- 大数据量的表必须建索引
- 避免 `SELECT *`，明确指定列
- 复杂查询优先使用 LINQ 聚合

## 5. 迁移规范

```bash
# 创建迁移
dotnet ef migrations add InitSysModule --context SysDbContext -o Infrastructure/Migrations

# 应用迁移 (开发环境)
dotnet ef database update --context SysDbContext
```

迁移文件命名: `{YYYYMMDD_HHMMSS}_{描述}.cs`
