# 前端技术栈选型

## 1. 双轨策略

继承 JeeSite 5 的设计理念: **一套后端同时支撑两套前端**。

```
┌──────────────────────────────────────────────────┐
│                    后端服务                         │
│            JeeSiteNET.Web.Api                     │
│         REST API + SignalR + JWT                  │
└──────────┬──────────────────────┬─────────────────┘
           │                      │
           ▼                      ▼
┌──────────────────┐   ┌──────────────────────────┐
│   分离版 (Vue 3)   │   │   全栈版 (Blazor/Razor)   │
│  SPA 单页应用      │   │  SSR 服务端渲染           │
│  适合复杂交互       │   │  适合内容型页面            │
└──────────────────┘   └──────────────────────────┘
```

## 2. 分离版技术栈 (Vue 3)

| 组件 | 选择 | 说明 |
|---|---|---|
| 框架 | Vue 3.5 | Composition API |
| 构建 | Vite 6 | 极速 HMR |
| UI 库 | Ant Design Vue 4 | 企业级组件库 |
| 语言 | TypeScript 5 | 类型安全 |
| 状态管理 | Pinia | 官方推荐 |
| 路由 | Vue Router 4 | SPA 路由 |
| HTTP | Axios | 请求拦截封装 |
| 管理模板 | Vue Vben Admin | 开箱即用 |
| 样式 | Less / UnoCSS | 动态样式 |
| 国际化 | vue-i18n | 多语言 |
| 图表 | ECharts + Vue-ECharts | 数据可视化 |
| 代码规范 | ESLint + Prettier + Stylelint | 统一格式 |
| 包管理 | pnpm | 快速、磁盘高效 |

### Vue Vben Admin 选型理由

JeeSite 分离版基于 Vben Admin 封装，我们沿用此方案：

- 开箱即用的后台管理模板
- 权限管理系统集成
- 多主题、多布局
- 国际化支持
- Table/Form 数据驱动视图
- 自定义插槽、渲染、事件

## 3. 全栈版技术栈 (Blazor/Razor)

| 组件 | 选择 | 说明 |
|---|---|---|
| 框架 | Blazor Server / WASM | 全栈 .NET |
| UI 库 | Bootstrap 5 | 经典响应式 |
| 模板引擎 | Razor Pages | SSR |
| 交互 | Blazor Server | 实时 UI 更新 |
| 兼容 | IE11+ | 政企场景 |
| 静态资源 | AdminLTE 适配 | JeeSite 风格 |

## 4. 核心组件封装

### BasicTable 表格组件

```vue
<template>
  <BasicTable
    :columns="columns"
    :request="fetchList"
    :page-request="pageRequest"
    @register="registerTable"
  />
</template>

<script setup lang="ts">
const columns: BasicColumn[] = [
  { title: '用户名', dataIndex: 'userName', align: 'left' },
  { title: '登录名', dataIndex: 'loginCode' },
  { title: '手机号', dataIndex: 'phone', width: 120 },
  { title: '邮箱', dataIndex: 'email', width: 200 },
  { title: '状态', dataIndex: 'status', align: 'center' },
  { title: '操作', dataIndex: 'action', width: 200 },
];

const [registerTable, { reload }] = useTable();

async function fetchList(params: PageRequest) {
  return await http.post('/api/v1/sys/user/list', params);
}
</script>
```

### BasicForm 表单组件

```vue
<template>
  <BasicForm @register="registerForm" @submit="handleSubmit" />
</template>

<script setup lang="ts">
const schemas: FormSchema[] = [
  { field: 'userName', label: '用户名称', component: 'Input', required: true },
  { field: 'userType', label: '用户类型', component: 'Select',
    componentProps: { options: userTypeOptions }, required: true },
  { field: 'email', label: '邮箱', component: 'Input',
    componentProps: { type: 'email' },
    rules: [{ type: 'email', message: '请输入正确的邮箱' }] },
];

const [registerForm, { resetFields }] = useForm({
  schemas,
  labelWidth: 100,
});
</script>
```

## 6. 前端开发规范

### 文件命名

- 组件: PascalCase (`UserTable.vue`, `RoleForm.vue`)
- 脚本: camelCase (`userService.ts`, `useTable.ts`)
- 目录: kebab-case (`user-management/`)

### 代码风格

- 使用 Composition API + `<script setup>`
- TypeScript 严格模式
- 组件默认导出，类型定义另建 `.d.ts` 或 `.type.ts`
