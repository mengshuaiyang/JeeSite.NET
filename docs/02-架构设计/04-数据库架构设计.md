# 数据库架构设计

## 1. 设计原则

继承 JeeSite 5 的数据库设计理念：

- **主键**: 有意义的编码，便于运维和识别（非 UUID）
- **表前缀**: `{模块前缀}_{表名}`，如 `Sys_User`
- **树结构**: 辅助字段体系，高效查询
- **审计字段**: 每张业务表包含 `create_by/create_date/update_by/update_date`
- **软删除**: 逻辑删除而非物理删除
- **冗余设计**: 业务表冗余基础表核心字段，减少关联查询

### 2.1 组织架构

#### Sys_Organization (组织机构表)

```sql
CREATE TABLE Sys_Organization (
    org_code        VARCHAR(100)    NOT NULL,       -- 机构编码 (主键)
    org_name        VARCHAR(200)    NOT NULL,       -- 机构名称
    parent_code     VARCHAR(100)    NOT NULL DEFAULT '0', -- 上级编码
    parent_codes    VARCHAR(2000)   NOT NULL,       -- 所有上级编码
    tree_sort       DECIMAL(10)     NOT NULL,       -- 本级排序
    tree_sorts      VARCHAR(2000)   NOT NULL,       -- 完整排序
    tree_leaf       CHAR(1)         NOT NULL DEFAULT '1', -- 是否叶子节点
    tree_level      DECIMAL(4)      NOT NULL DEFAULT 0,   -- 层次级别
    tree_names      VARCHAR(2000)   NOT NULL,       -- 全路径名称
    org_type        VARCHAR(100),                   -- 机构类型
    org_type_name   VARCHAR(200),                   -- 机构类型名称
    status          CHAR(1)         NOT NULL DEFAULT '0', -- 状态(0正常1删除2停用)
    create_by       VARCHAR(100),                   -- 创建者
    create_date     DATETIME,                       -- 创建时间
    update_by       VARCHAR(100),                   -- 更新者
    update_date     DATETIME,                       -- 更新时间
    remarks         VARCHAR(500),                   -- 备注
    PRIMARY KEY (org_code)
);
CREATE INDEX idx_org_parent ON Sys_Organization(parent_code);
CREATE INDEX idx_org_parent_codes ON Sys_Organization(parent_codes);
CREATE INDEX idx_org_tree_sorts ON Sys_Organization(tree_sorts);
```

## 3. 树结构辅助字段机制

### 3.1 工作原理

### 3.2 辅助字段自动维护

使用 EF Core 的 `SaveChangesInterceptor` 在保存时自动维护:

```csharp
public class TreeEntityInterceptor : SaveChangesInterceptor
{
    public override InterceptionResult<int> SavingChanges(
        DbContextEventData eventData, InterceptionResult<int> result)
    {
        var context = eventData.Context;
        var entries = context.ChangeTracker.Entries<ITreeEntity>();

        foreach (var entry in entries)
        {
            if (entry.State == EntityState.Added || entry.State == EntityState.Modified)
            {
                var entity = entry.Entity;
                var parent = context.Find(entity.GetType(), entity.ParentCode) as ITreeEntity;

                // 自动计算辅助字段
                entity.TreeLevel = (parent?.TreeLevel ?? -1) + 1;
                entity.ParentCodes = parent != null
                    ? parent.ParentCodes + entity.ParentCode + ","
                    : "";
                entity.TreeNames = parent != null
                    ? parent.TreeNames + "/" + entity.GetName()
                    : entity.GetName();

                // 级联更新子节点
                if (entry.State == EntityState.Modified)
                {
                    CascadeUpdateChildren(context, entity);
                }
            }
        }

        return base.SavingChanges(eventData, result);
    }
}
```

## 5. 多租户方案

支持三种隔离级别，通过配置切换:

```json
{
  "MultiTenant": {
    "Enabled": true,
    "IsolationMode": "SharedTable" // SharedTable | Schema | Database
  }
}
```

| 模式 | 说明 | 适用场景 |
|---|---|---|
| SharedTable | 共用表 + TenantCode 字段 | 中小型SaaS |
| Schema | 同库不同 Schema (dbo/tenant1/) | 中型企业 |
| Database | 独立数据库 | 大型/高安全要求 |

## 6. 数据库迁移策略

### 6.1 EF Core Migration 方案

```
┌──────────────────────────────────────────────┐
│  启动时 (开发环境)                               │
│  context.Database.MigrateAsync()               │
│  → 自动执行所有待处理 Migration                  │
├──────────────────────────────────────────────┤
│  生产环境                                       │
│  dotnet ef database update                     │
│  → 运维手动执行 / CI/CD 自动执行                  │
├──────────────────────────────────────────────┤
│  模块升级 (JeeSite特有)                          │
│  每个模块的 Migration 独立管理                    │
│  通过 Sys_Module 表记录模块数据库版本              │
│  按版本号执行增量 Migration 脚本                  │
└──────────────────────────────────────────────┘
```

### 6.2 目录结构

```
src/JeeSiteNET.Infrastructure/EntityFrameworkCore/Migrations/

├── {timestamp}_AddUserEmail.cs            # 业务演进

```

modules/JeeSiteNET.Modules.Bpm/Infrastructure/Migrations/
