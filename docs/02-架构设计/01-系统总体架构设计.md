# 系统总体架构设计

## 1. 架构设计原则

继承 JeeSite 5 的核心设计哲学，结合 .NET 10 生态最佳实践:

| 原则 | 说明 |
|---|---|
| **微内核** | 核心层最小化，仅包含基础设施框架，所有功能以模块形式扩展 |
| **松耦合** | 模块间通过接口依赖，不允许直接引用实现 |
| **约定优于配置** | 标准化的命名/目录/分层，降低认知负荷 |
| **封装但不限制** | 提供便捷的封装 API，同时开放原生扩展点 |
| **不侵入** | 框架代码不侵入业务代码，升级无忧 |
| **低代码能力** | 在线建表 + 代码生成，减少70%重复编码 |

## 3. 请求处理流程

### 3.1 前后分离版 (Vue 3 + Web API)

```
Vue 前端 ──HTTP──▶ API Gateway ──▶ Controller ──▶ Service ──▶ Repository ──▶ Database
                        │               │            │
                        │               ├─ 权限验证   ├─ 数据权限过滤
                        │               ├─ 参数校验   ├─ 缓存查询(L1→L2)
                        │               └─ 日志记录   └─ 事务管理
                        │
                        └──▶ JWT 认证 ──▶ Identity ──▶ 角色/权限
```

### 3.2 全栈版 (Razor Pages + MVC)

```
浏览器 ──▶ ASP.NET Core MVC ──▶ Controller ──▶ Service ──▶ Repository ──▶ Database
                │                    │             │
                ├─ Razor Pages SSR   ├─ 同API版    ├─ 同API版
                ├─ Session/Cookie    │
                └─ Anti-Forgery      │
                                     └─ 可同时为 Vue 后端 API
```

## 4. 模块间依赖关系

```
JeeSiteNET.Core        ← 无依赖，纯工具
       ↑
JeeSiteNET.Infrastructure ← 依赖 Core
       ↑
JeeSiteNET.Modules.Sys    ← 依赖 Core + Infrastructure
       ↑
JeeSiteNET.Modules.Bpm    ← 依赖 Core + Infrastructure + Sys (可选)
JeeSiteNET.Modules.Cms    ← 依赖 Core + Infrastructure + Sys (可选)
       ↑
JeeSiteNET.Web.Api        ← 依赖所有引用模块
JeeSiteNET.Web.Host       ← 依赖所有引用模块
```

## 5. 关键技术决策

| 决策 | 选择 | 理由 |
|---|---|---|
| 模块加载 | 编译时引用 | 第一阶段简单可靠，二期可升级为动态加载 |
| 前端双轨 | Vue3 + Blazor 同时支持 | 继承 JeeSite 一套后端支撑两套前端理念 |
| ORM 增强 | EF Core Fluent API + 自研 EntityBuilder | 比注解更灵活，编译期检查 |
| 缓存 | FusionCache | 官方推荐的 L1+L2 方案，类比 J2Cache |
| 工作流 | Elsa Workflows 3.x | .NET 生态最活跃的工作流引擎 |
| API 文档 | Scalar + OpenAPI 3.1 | 现代美观的 API 文档 |
