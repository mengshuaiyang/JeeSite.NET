# 分层架构设计

## 1. 分层概述

JeeSite.NET 采用六层架构，严格遵循依赖倒置原则:

```
┌─────────────────────────────────────────────────────┐
│  1. 表现层 (Presentation Layer)                       │
│  - Web API (RESTful)                                 │
│  - Web Host (MVC + Razor Pages)                      │
│  - 依赖: Application Layer                           │
├─────────────────────────────────────────────────────┤
│  2. 应用层 (Application Layer)                        │
│  - 应用服务、DTO、CQRS 命令/查询                      │
│  - 依赖: Domain Layer                                │
├─────────────────────────────────────────────────────┤
│  3. 领域层 (Domain Layer)                             │
│  - 实体、聚合、值对象、领域事件、仓储接口                │
│  - 零外部依赖 (仅 .NET Base Class Library)            │
├─────────────────────────────────────────────────────┤
│  4. 基础设施层 (Infrastructure Layer)                  │
│  - 仓储实现、EF Core、缓存、消息队列、认证              │
│  - 依赖: Domain Layer                                │
├─────────────────────────────────────────────────────┤
│  5. 核心层 (Core Layer)                               │
│  - 基础工具类、统一响应、分页基类                       │
│  - 被所有层引用 (纯抽象/工具，无业务逻辑)               │
├─────────────────────────────────────────────────────┤
│  6. 模块层 (Module Layer)                              │
│  - 业务模块，每个模块内部有独立的 Domain/App/Infra      │
│  - 依赖: Core + Infrastructure                        │
└─────────────────────────────────────────────────────┘
```

## 2. 各层详细设计

### 2.1 核心层 (Core Layer)

**职责**: 提供不依赖任何第三方的基础抽象和工具类

**关键类**:

```
JeeSiteNET.Core/
├── ApiResult.cs                  # 统一响应: ApiResult<T> { Code, Message, Data }
├── ApiResultStatus.cs            # 响应状态码枚举
├── PageRequest.cs                # 分页请求基类
├── PageResult.cs                 # 分页响应
├── BaseEntity.cs                 # 基础实体: Id/CreateBy/CreateDate/UpdateBy/UpdateDate
├── DataEntity.cs                 # 业务数据实体: Status/Remarks
├── TreeEntity.cs                 # 树表实体: ParentCode/ParentCodes/TreeSort/TreeSorts/TreeLeaf/TreeLevel/TreeNames
├── IRepository.cs                # 泛型仓储接口
├── ICrudService.cs               # 泛型CRUD服务接口
├── QueryType.cs                  # 查询类型枚举 (LIKE/EQ/BETWEEN等)
├── SqlMap.cs                     # 查询条件构造器 (类比JeeSite SqlMap)
├── Modules/
│   ├── IModuleDescriptor.cs      # 模块描述接口
│   ├── IModuleInstaller.cs        # 模块安装器接口
│   └── ModuleLoader.cs           # 模块加载器 (扫描、注册)
├── Security/
│   ├── ICurrentUser.cs           # 当前用户上下文
│   └── IDataScope.cs             # 数据权限接口
└── Utils/
    ├── IdGenerator.cs            # 有意义的主键生成
    ├── StringUtils.cs            # 字符串工具
    ├── ReflectUtils.cs           # 反射工具
    └── ExcelUtils.cs             # Excel导入导出
```

### 2.2 领域层 (Domain Layer)

**职责**: 核心业务逻辑，零外部依赖

每个模块内的 Domain 结构:

```
JeeSiteNET.Modules.Sys.Domain/
├── Entities/
│   ├── User.cs                   # 用户实体
│   ├── Role.cs                   # 角色实体
│   ├── Menu.cs                   # 菜单实体 (树)
│   ├── Organization.cs           # 组织机构实体 (树)
│   ├── DictData.cs               # 字典数据
│   └── ...
├── Aggregates/
│   └── UserAggregate.cs          # 用户聚合 (可选)
├── ValueObjects/
│   ├── Address.cs                # 地址值对象
│   └── PhoneNumber.cs            # 电话号码
├── Events/
│   ├── UserCreatedEvent.cs       # 领域事件
│   └── UserLoggedInEvent.cs
└── Interfaces/
    ├── IUserRepository.cs        # 仓储接口
    └── IRoleRepository.cs
```

### 2.3 应用层 (Application Layer)

**职责**: 用例编排、DTO转换、事务协调

```
JeeSiteNET.Modules.Sys.Application/
├── Services/
│   ├── UserService.cs            # 用户应用服务
│   ├── RoleService.cs            # 角色应用服务
│   └── MenuService.cs            # 菜单应用服务
├── DTOs/
│   ├── UserDto.cs                # 用户DTO
│   ├── UserCreateDto.cs          # 创建用户DTO
│   ├── UserUpdateDto.cs          # 更新用户DTO
│   └── UserPageRequest.cs        # 用户分页请求
├── CQRS/                         # (可选 CQRS 模式)
│   ├── Commands/
│   └── Queries/
├── Mappings/
│   └── UserProfile.cs            # AutoMapper/Manual配置
└── Validators/
    └── UserCreateValidator.cs    # FluentValidation
```

## 3. 数据流转示例

以"用户分页查询"为例:

```
1. 请求: GET /api/v1/sys/user?pageNo=1&pageSize=10&userName=张
       │
2. Controller UserController.List(PageRequest<UserDto> request)
       │  ├─ 参数校验 (FluentValidation)
       │  ├─ 权限验证 ([Permission("sys:user:list")])
       │  └─ 调用 Service
       ▼
3. UserService.FindPageAsync(request)
       │  ├─ 应用数据权限 (注入 dataScope SQL)
       │  ├─ 查询缓存 (FusionCache TryGet)
       │  ├─ 缓存未命中 → 调用 Repository
       │  └─ 设置缓存
       ▼
4. UserRepository.FindPageAsync(request)
       │  ├─ EF Core IQueryable 构建
       │  ├─ 应用 Global Query Filters
       │  │   ├─ 软删除过滤 (delFlag = 0)
       │  │   ├─ 多租户过滤 (tenantId)
       │  │   └─ 数据权限过滤 (dataScope)
       │  ├─ 排序 + 分页 (OrderBy + Skip/Take)
       │  └─ 执行 SQL → 数据库
       ▼
5. 返回: PageResult<UserDto> { Total, PageNo, PageSize, List }
```

## 4. 分层约束

| 规则 | 强制方式 |
|---|---|
| Domain 层不能引用任何其他层 | 项目引用检查 + ArchUnitNET 测试 |
| Application 层只能引用 Domain + Core | 同上 |
| Infrastructure 层可引用 Domain + Core | 但仓储实现必须面向接口 |
| Web 层可引用所有层 | 但只能通过 Application 服务 |
| Core 层不能引用任何业务项目 | 纯工具类和抽象 |
