# 模块化架构设计

## 1. 设计目标

参考 JeeSite 5 的模块化设计理念，在 .NET 生态中实现:

- **编译时模块**: 第一阶段采用项目引用方式，简单可靠
- **标准化模块结构**: 每个模块有统一的目录规范和接口
- **模块自注册**: 通过 `IModuleInstaller` 自动注册服务和配置
- **可选加载**: Web 项目通过引用决定包含哪些模块
- **未来可升级**: 预留动态加载接口，后续可升级为 `AssemblyLoadContext` 插件模式

## 2. 模块生命周期

```
┌────────────────┐
│   模块开发       │  开发者按模板创建模块
│  dotnet new     │
└───────┬────────┘
        ▼
┌────────────────┐
│   模块编译       │  dotnet build → DLL
│  NuGet 包      │  或项目引用
└───────┬────────┘
        ▼
┌────────────────┐
│  Web引用模块     │  Web.csproj 添加 ProjectReference
│  编译时绑定      │
└───────┬────────┘
        ▼
┌────────────────┐
│  启动时注册      │  ModuleLoader 扫描 IModuleInstaller
│  DI 注册/迁移   │  注册服务、应用EF Core迁移
└───────┬────────┘
        ▼
┌────────────────┐
│   运行时使用      │  菜单/权限注册到 Sys_Module 表
│   可启停        │  通过模块管理界面启停
└────────────────┘
```

## 3. 模块接口定义

### 3.1 IModuleDescriptor — 模块元数据

```csharp
public interface IModuleDescriptor
{
    /// <summary>模块编码，如 "Sys", "Bpm"</summary>
    string Code { get; }
    /// <summary>模块名称，如 "系统管理模块"</summary>
    string Name { get; }
    /// <summary>模块版本</summary>
    string Version { get; }
    /// <summary>依赖的模块编码列表</summary>
    string[] Dependencies { get; }
    /// <summary>排序号</summary>
    int SortOrder { get; }
}
```

### 3.2 IModuleInstaller — 模块安装器

```csharp
public interface IModuleInstaller
{
    /// <summary>注册模块服务到 DI 容器</summary>
    void ConfigureServices(IServiceCollection services, IConfiguration configuration);

    /// <summary>配置应用程序中间件管道</summary>
    void ConfigureApplication(IApplicationBuilder app);

    /// <summary>配置数据库迁移</summary>
    void ConfigureDatabase(ModelBuilder modelBuilder);

    /// <summary>配置模块端点 (API 路由等)</summary>
    void ConfigureEndpoints(IEndpointRouteBuilder endpoints);
}
```

### 3.3 ModuleLoader — 模块加载器

```csharp
public class ModuleLoader
{
    private readonly List<IModuleDescriptor> _modules = new();

    /// <summary>扫描程序集并注册所有模块</summary>
    public void LoadModules(IServiceCollection services, IConfiguration configuration)
    {
        // 1. 扫描当前 AppDomain 的所有程序集
        var assemblies = AppDomain.CurrentDomain.GetAssemblies();

        // 2. 查找 IModuleInstaller 实现
        foreach (var assembly in assemblies)
        {
            var installerTypes = assembly.GetTypes()
                .Where(t => typeof(IModuleInstaller).IsAssignableFrom(t)
                            && t.IsClass && !t.IsAbstract);

            foreach (var type in installerTypes)
            {
                var installer = (IModuleInstaller)Activator.CreateInstance(type)!;
                installer.ConfigureServices(services, configuration);

                // 注册模块描述
                var descriptor = type.GetCustomAttribute<ModuleDescriptionAttribute>();
                if (descriptor != null)
                {
                    _modules.Add(descriptor);
                }
            }
        }
    }

    /// <summary>应用所有模块的数据库配置</summary>
    public void ConfigureDatabase(ModelBuilder modelBuilder)
    {
        // 调用每个模块的 ConfigureDatabase
    }
}
```

## 4. 模块目录结构规范

每个模块必须遵循以下目录结构:

```
JeeSiteNET.Modules.{ModuleName}/
├── Domain/
│   ├── Entities/
│   │   └── {EntityName}.cs
│   ├── Enums/
│   │   └── {EnumName}.cs
│   └── Interfaces/
│       └── I{EntityName}Repository.cs
├── Application/
│   ├── Services/
│   │   └── {EntityName}Service.cs
│   ├── DTOs/
│   │   ├── {EntityName}Dto.cs
│   │   └── {EntityName}PageRequest.cs
│   └── Validators/
│       └── {EntityName}Validator.cs
├── Infrastructure/
│   ├── EntityConfigurations/
│   │   └── {EntityName}Configuration.cs
│   └── Repositories/
│       └── {EntityName}Repository.cs
├── Controllers/
│   └── {EntityName}Controller.cs
├── ModuleInstaller.cs              # 实现 IModuleInstaller
└── {ModuleName}Module.cs           # 模块入口 (属性标记)
```

## 5. 核心模块清单 (第一阶段)

| 模块编码 | 模块名称 | 依赖 | 说明 |
|---|---|---|---|
| Core | 核心层 | 无 | 基础工具、抽象基类、模块框架 |
| Sys | 系统管理 | Core | 组织机构、用户、角色、权限、菜单、字典 |
| Bpm | 工作流 | Core, Sys | 流程定义、实例管理、任务办理 |
| Cms | 内容管理 | Core, Sys | 站点、栏目、文章发布 |
| File | 文件管理 | Core, Sys | 文件上传、预览、分享 |

## 6. 模块间的数据隔离

### 6.1 数据库表命名

```
{ModulePrefix}_{TableName}
```

| 模块 | 前缀 | 示例 |
|---|---|---|
| Sys | Sys_ | Sys_User, Sys_Role, Sys_Menu |
| Bpm | Bpm_ | Bpm_Process, Bpm_Task |
| Cms | Cms_ | Cms_Site, Cms_Article |
| File | File_ | File_Upload, File_Share |

### 6.2 Schema 分离 (可选)

```csharp
// 按模块使用不同 Schema
builder.Entity<User>().ToTable("User", schema: "sys");
builder.Entity<Process>().ToTable("Process", schema: "bpm");
```

## 7. 编译时模块 vs 运行时插件

| 特性 | 编译时模块 (第一阶段) | 运行时插件 (未来升级) |
|---|---|---|
| 实现方式 | ProjectReference | AssemblyLoadContext |
| 模块发现 | 编译期确定 | 扫描 /plugins 目录 |
| 版本隔离 | 同一版本 | 可独立版本 |
| 热加载 | 不支持 | 支持 (collectible ALC) |
| 复杂度 | 低 | 高 |
| 适用场景 | 核心模块 | 第三方扩展模块 |

## 8. 模块开发示例

### 创建新模块

```xml
<!-- JeeSiteNET.Modules.Bpm.csproj -->
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net10.0</TargetFramework>
    <RootNamespace>JeeSiteNET.Modules.Bpm</RootNamespace>
  </PropertyGroup>
  <ItemGroup>
    <ProjectReference Include="..\..\src\JeeSiteNET.Core\JeeSiteNET.Core.csproj" />
    <ProjectReference Include="..\..\src\JeeSiteNET.Infrastructure\JeeSiteNET.Infrastructure.csproj" />
  </ItemGroup>
</Project>
```

### 模块安装器实现

```csharp
[ModuleDescription(Code = "Bpm", Name = "工作流模块", Version = "1.0.0",
    Dependencies = new[] { "Sys" })]
public class BpmModuleInstaller : IModuleInstaller
{
    public void ConfigureServices(IServiceCollection services, IConfiguration configuration)
    {
        // 注册 BPM 服务
        services.AddScoped<IProcessService, ProcessService>();
        services.AddScoped<ITaskService, TaskService>();

        // 注册 BPM DbContext
        services.AddDbContext<BpmDbContext>(options =>
            options.UseSqlServer(configuration.GetConnectionString("DefaultConnection")));
    }

    public void ConfigureApplication(IApplicationBuilder app) { }

    public void ConfigureDatabase(ModelBuilder modelBuilder)
    {
        modelBuilder.ApplyConfiguration(new ProcessConfiguration());
        modelBuilder.ApplyConfiguration(new TaskConfiguration());
    }

    public void ConfigureEndpoints(IEndpointRouteBuilder endpoints)
    {
        endpoints.MapControllerRoute("bpm_default", "api/v1/bpm/{controller}/{action}");
    }
}
```
