# 多租户与数据权限设计

## 1. 多租户架构

### 1.1 隔离模式

支持三种隔离级别，继承 JeeSite 5 的 SaaS 多租户方案:

```
┌───────────────────────────────────────────────┐
│              多租户隔离模式                       │
├───────────────────┬───────────────────────────┤
│   模式             │           描述              │
├───────────────────┼───────────────────────────┤
│   SharedTable     │ 共用表 + TenantCode 列     │
│   Schema          │ 同库不同 Schema             │
│   Database        │ 独立数据库                  │
└───────────────────┴───────────────────────────┘
```

### 1.2 SharedTable 模式 (默认)

所有租户共享数据库表，通过 `tenant_code` 字段隔离:

```sql
-- 每张业务表增加租户字段
ALTER TABLE Sys_User ADD tenant_code VARCHAR(100) NOT NULL DEFAULT '';

CREATE INDEX idx_user_tenant ON Sys_User(tenant_code);
```

EF Core Global Query Filter 实现:

```csharp
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    if (_multiTenantOptions.Enabled)
    {
        // 为所有实体自动添加租户过滤
        foreach (var entity in modelBuilder.Model.GetEntityTypes())
        {
            if (entity.ClrType.IsAssignableTo(typeof(IMultiTenant)))
            {
                var parameter = Expression.Parameter(entity.ClrType, "e");
                var property = Expression.Property(parameter, "TenantCode");
                var value = Expression.Constant(_currentTenant.TenantCode);
                var lambda = Expression.Lambda(
                    Expression.Equal(property, value), parameter);

                entity.SetQueryFilter(lambda);
            }
        }
    }
}
```

### 1.3 Schema 模式

不同租户使用同一数据库的不同 Schema:

```csharp
// 根据租户动态设置 Schema
builder.Entity<User>().ToTable("User", schema: $"tenant_{_currentTenant.TenantCode}");
```

### 1.4 Database 模式

完全隔离的独立数据库:

```csharp
// 每个租户动态创建/切换数据库连接
services.AddDbContext<SysDbContext>((sp, options) =>
{
    var tenant = sp.GetRequiredService<ICurrentTenant>();

```

### 1.5 租户解析策略

```csharp
public interface ITenantResolver
{
    string ResolveTenantCode(HttpContext context);
}

public class HostTenantResolver : ITenantResolver
{
    public string ResolveTenantCode(HttpContext context)
    {
        // 1. 从域名解析: tenant1.example.com
        var host = context.Request.Host.Host;
        var parts = host.Split('.');
        if (parts.Length >= 2) return parts[0];

        // 2. 从 Header 解析
        var header = context.Request.Headers["X-Tenant-Code"].FirstOrDefault();
        if (header != null) return header;

        // 3. 从 JWT Claim 解析
        var claim = context.User?.FindFirst("tenant_code")?.Value;
        if (claim != null) return claim;

        return "default";
    }
}
```

## 2. 数据权限设计

### 2.1 三层权限模型

```
┌─────────────────────────────────────────────────┐
│  第一层: 身份认证 (Authentication)                  │
│  用户登录 → JWT Token → 身份确认                   │
├─────────────────────────────────────────────────┤
│  第二层: 功能权限 (Functional Permission)           │
│  角色 → 菜单/按钮 → 用户能访问什么功能              │
│  实现: [Permission("sys:user:edit")] Policy        │
├─────────────────────────────────────────────────┤
│  第三层: 数据权限 (Data Permission)                 │
│  用户能看到哪些数据行                               │
│  实现: EF Core Query Filter / SQL 注入             │
└─────────────────────────────────────────────────┘
```

### 2.2 数据权限范围类型

| 范围类型 | 说明 | SQL 过滤条件 |
|---|---|---|
| `all` | 全部数据 | 无过滤 |
| `company` | 本公司 | `org_codes LIKE '%,companyCode,%'` |
| `company_and_child` | 本公司及下属 | `org_codes LIKE '%,companyCode%'` |
| `office` | 本部门 | `org_code = 'currentOrg'` |
| `office_and_child` | 本部门及下属 | `parent_codes LIKE '%,currentOrg,%'` |
| `self` | 仅本人 | `create_by = 'currentUser'` |
| `custom` | 自定义 SQL | 由 `scope_custom_sql` 定义 |

### 2.3 数据权限规则配置

```csharp
// 数据权限规则实体
public class DataPermissionRule
{
    public string PermissionId { get; set; }
    public string RoleCode { get; set; }       // 角色编码
    public string UserCode { get; set; }       // 用户编码 (可选)
    public string TargetType { get; set; }     // 目标类型 (org/company/custom)
    public string TargetValue { get; set; }    // 目标值
    public string ScopeType { get; set; }      // 范围类型
    public string ScopeCustomSql { get; set; } // 自定义SQL

    // 拥有的权限 vs 管理的权限
    // 'own' - 拥有的权限 (处理自己的业务数据)
    // 'manage' - 管理的权限 (管理其他用户/组织)
    public string PermissionCategory { get; set; }
}
```

### 2.4 数据权限注入实现

```csharp
public interface IDataScopeService
{
    /// <summary>
    /// 获取数据权限 SQL 条件
    /// </summary>
    /// <param name="targetType">数据类型: user/org/role 等</param>
    /// <returns>SQL WHERE 条件</returns>
    string GetDataScopeSql(string targetType);

    /// <summary>
    /// 应用数据权限到 IQueryable
    /// </summary>
    IQueryable<T> ApplyDataScope<T>(IQueryable<T> query, string targetType)
        where T : class;
}

public class DataScopeService : IDataScopeService
{
    private readonly ICurrentUser _currentUser;

    public IQueryable<T> ApplyDataScope<T>(IQueryable<T> query, string targetType)
    {
        var rules = GetUserDataPermissionRules(targetType);

        // 组合所有规则 (OR关系)
        Expression? combined = null;
        var parameter = Expression.Parameter(typeof(T), "e");

        foreach (var rule in rules)
        {
            var expression = BuildScopeExpression<T>(parameter, rule);
            combined = combined == null
                ? expression
                : Expression.OrElse(combined, expression);
        }

        if (combined != null)
        {
            var lambda = Expression.Lambda<Func<T, bool>>(combined, parameter);
            query = query.Where(lambda);
        }

        return query;
    }

    private Expression BuildScopeExpression<T>(ParameterExpression parameter, DataPermissionRule rule)
    {
        return rule.ScopeType switch
        {
            "all" => Expression.Constant(true),
            "self" => Expression.Equal(
                Expression.Property(parameter, "CreateBy"),
                Expression.Constant(_currentUser.UserCode)),
            "office" => Expression.Equal(
                Expression.Property(parameter, "OrgCode"),
                Expression.Constant(_currentUser.OrgCode)),
            "office_and_child" => Expression.Call(
                typeof(EF).GetMethod(nameof(EF.Functions))!,
                typeof(DbFunctionsExtensions).GetMethod("Like")!,
                Expression.Property(parameter, "ParentCodes"),
                Expression.Constant($"%,{_currentUser.OrgCode},%")),
            _ => Expression.Constant(true)
        };
    }

    private List<DataPermissionRule> GetUserDataPermissionRules(string targetType)
    {
        // 从缓存/数据库获取当前用户的数据权限规则
        // 规则来源: 角色关联的规则 + 用户直接关联的规则
        return _cache.GetOrSet($"data_perm:{_currentUser.UserCode}:{targetType}",
            () => _dbContext.DataPermissionRules
                .Where(r => (r.RoleCode == _currentUser.RoleCodes.Contains(r.RoleCode)
                            || r.UserCode == _currentUser.UserCode)
                            && r.TargetType == targetType)
                .ToList());
    }
}
```

### 2.5 管理权限 vs 拥有权限

JeeSite 5 特有的权限分类:

```
拥有的权限 (Own Permission):
  - 用户处理自己业务数据时的权限
  - 如: 只能查看本部门的订单

管理的权限 (Manage Permission):
  - 管理员管理其他用户的权限
  - 如: 人事经理可以管理本部门的所有用户
  - 适用于: 用户管理、机构管理等功能

配置:
  appsettings.json:
    "User": {
      "AdminCtrlPermi": "0"  // 0=管理的权限 1=拥有的权限
    }
```

### 2.6 防越级授权

```csharp
// 分配角色时检查
public async Task AssignRoleAsync(string targetUserCode, List<string> roleCodes)
{
    var currentUser = await _currentUserService.GetCurrentUserAsync();

    foreach (var roleCode in roleCodes)
    {
        var role = await _roleRepository.GetAsync(roleCode);

        // 不能分配比自己级别高的角色
        if (role.Weight > currentUser.MaxRoleWeight)
        {
            throw new UnauthorizedException($"不能分配 [{role.RoleName}] 角色，权限不足");
        }

        // 不能分配自己没有的角色
        if (!currentUser.RoleCodes.Contains(roleCode) && !currentUser.IsSuperAdmin)
        {
            throw new UnauthorizedException($"无权分配 [{role.RoleName}] 角色");
        }
    }

    // 执行分配
    await _userRoleRepository.AssignRolesAsync(targetUserCode, roleCodes);
}
```

public async Task<List<UserPermissionInfo>> AuditMenuPermissionAsync(string menuCode)

## 4. EF Core Global Query Filter vs 手动注入

| 方式 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| Global Query Filter | 自动、无侵入 | 静态、不能按需关闭 | 软删除、多租户(SharedTable) |
| Interceptor SQL 注入 | 灵活、动态 | 手动实现 | 数据权限(规则可变) |
| Repository 层过滤 | 明确可控 | 每处都要写 | 简单权限 |
