## 2026-06-12 安全加固 & 国密标准实现更新

**A. SM2/SM4 国密标准实现（BouncyCastle）**

| 项目 | 原实现（简化版） | 新实现（标准） |
|------|-----------------|----------------|
| SM2 签名 | 使用 nistP256 曲线 + SHA256（非国密） | 使用 BouncyCastle sm2p256v1 曲线 + SM3 摘要 |
| SM2 加密 | 未实现（依赖 AES 对称） | SM2Engine 椭圆曲线加密 C1C2C3 DER |
| SM4 加密 | 手写 S-box（uint[] 越界风险） | BouncyCastle SM4Engine + CBC + PKCS7 |
| SM3 哈希 | 已实现 | 已实现（未变更） |

**B. 安全防护工具类（新增）**

| 工具类 | 文件 | 功能 |
|--------|------|------|
| HtmlSanitizerUtil | src/JeeSiteNET.Core/Utils/HtmlSanitizerUtil.cs | HTML 富文本 XSS 清洗（白名单标签/属性，双重解码防护） |
| FileSecurityUtil | src/JeeSiteNET.Core/Utils/FileSecurityUtil.cs | 文件扩展名白名单、文件签名校验、路径遍历检测 |

**C. 安全中间件与路由守卫**

| 位置 | 文件 | 增强内容 |
|------|------|---------|
| 响应头 | src/JeeSiteNET.Web.Api/Middleware/SecurityHeadersMiddleware.cs | 添加 Content-Security-Policy、Permissions-Policy 头 |
| API 路由 | src/JeeSiteNET.Core/.../router/index.ts | beforeEach 加入 [Permission] 特性路由守卫检测，无权限重定向 |
| 文件上传 | modules/JeeSiteNET.Modules.Sys/Controllers/FileController.cs | FileService 加入 FileSecurityUtil 扩展名/签名校验 |
| UEditor | src/JeeSiteNET.Core/UEditor/UEditorActionHandler.cs | 加入文件签名校验 + SSRF 防护（禁止内网 IP） |

**D. 前端按钮级权限**

| 组件 | 文件 | 说明 |
|------|------|------|
| v-permission 指令 | frontend/src/directives/permission.ts | 使用指令 v-permission="'cms:article:add'"，无权限时从 DOM 中移除 |
| usePermission() composable | frontend/src/composables/usePermission.ts | has() / hasAny() / hasAll() / getAll() |

**E. ArticleService HTML 清洗**

在 modules/JeeSiteNET.Modules.Cms/Application/Services/ArticleService.cs 的 SaveAsync 中，对 Title/Subtitle/Summary/Author/Image/Tags 字段调用 HtmlSanitizerUtil.SanitizeStrict()，对 content 字段调用 SanitizeRich()，确保富文本不含 script/iframe/onclick 等危险元素。
