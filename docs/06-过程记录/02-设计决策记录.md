# 设计决策记录 (ADR)

> Architecture Decision Records — 记录所有关键架构决策及其上下文。

## ADR-001: 采用 .NET 10 LTS

- **日期**: 2026-06-06
- **状态**: ✅ 已采纳
- **上下文**: 需要选择一个 .NET 版本作为基础
- **决策**: 使用 .NET 10 LTS (2026年11月发布)
- **理由**: LTS 版本长期支持，C# 14 最新语言特性
- **替代方案**: .NET 8 LTS (稳定但特性较旧), .NET 9 (非LTS)
- **影响**: 需使用 .NET 10 SDK 构建

## ADR-002: 编译时模块 (第一阶段)

- **日期**: 2026-06-06
- **状态**: ✅ 已采纳
- **上下文**: 模块的加载方式选择
- **决策**: 第一阶段采用编译时引用 (ProjectReference)
- **理由**: 简单可靠，编译期检查，适合核心模块
- **未来计划**: 第二阶段可升级为 AssemblyLoadContext 动态加载
- **影响**: 模块增减需要修改 Web 项目的引用

## ADR-003: 前端双轨 (Vue 3 + Blazor)

- **日期**: 2026-06-06
- **状态**: ✅ 已采纳
- **上下文**: 前端技术栈选型
- **决策**: 同时支持 Vue 3 分离版 和 Blazor 全栈版
- **理由**: 继承 JeeSite "一套后端两套前端" 理念
- **Vue 3**: Vite + Ant Design Vue + TypeScript
- **Blazor**: .NET 10 Blazor Server + Bootstrap 5
- **影响**: 需维护两套前端代码

## ADR-004: EF Core + 自研 EntityBuilder

- **日期**: 2026-06-06
- **状态**: ✅ 已采纳
- **上下文**: ORM 框架选择及增强需求
- **决策**: 使用 EF Core 10 + 自研 EntityBuilder (Fluent API 增强)
- **理由**: EF Core 是官方ORM，Fluent API 比属性注解更灵活
- **类比**: JeeSite 的 @Table 注解 → EntityBuilder
- **影响**: 需开发 EntityBuilder 辅助类

## ADR-005: FusionCache 替代 J2Cache

- **日期**: 2026-06-06
- **状态**: ✅ 已采纳
- **上下文**: 缓存架构选型
- **决策**: 使用 FusionCache (Memory L1 + Redis L2)
- **理由**: 与 JeeSite J2Cache 概念完全对应，社区活跃
- **特性**: 防崩溃、回退、Backplane
- **影响**: 需封装统一 CacheUtils

## ADR-006: 树表辅助字段机制

- **日期**: 2026-06-06
- **状态**: ✅ 已采纳
- **上下文**: 树形数据存储方案
- **决策**: 继承 JeeSite 树表辅助字段体系 (7字段)
- **理由**: 已验证的成熟方案，查询性能极佳
- **实现**: EF Core SaveChangesInterceptor 自动维护
- **影响**: 所有树表实体需继承 TreeEntity

## ADR-007: 主键使用有意义编码

- **日期**: 2026-06-06
- **状态**: ✅ 已采纳
- **上下文**: 主键生成策略
- **决策**: 使用有意义的编码 (非 UUID/GUID)
- **理由**: 便于运维和识别，与 JeeSite 一致
- **示例**: 组织编码 370000, 角色编码 role_admin
- **影响**: 主键需要手动赋值或使用 IdGenerator

## ADR-008: 数据权限使用 EF Core Query Filter

- **日期**: 2026-06-06
- **状态**: ✅ 已采纳
- **上下文**: 数据行级别权限实现
- **决策**: EF Core Global Query Filter + 动态规则组合
- **理由**: 自动过滤，无侵入
- **范围**: 软删除 / 多租户 / 数据权限 均通过 Query Filter
- **影响**: 复杂规则组合需要 Expression 拼接

## ADR-009: Elsa Workflows 作为工作流引擎

- **日期**: 2026-06-06
- **状态**: ✅ 已采纳
- **上下文**: 工作流引擎选型
- **决策**: 使用 Elsa Workflows 3.x
- **理由**: .NET 生态最活跃的工作流引擎
- **特性**: 可视化设计器、BPMN 2.0、长/短运行
- **类比**: JeeSite 使用 Flowable
- **影响**: 需安装 Elsa Studio 前端

## ADR-010: API 路由 /api/v1/{module}/{controller}

- **日期**: 2026-06-06
- **状态**: ✅ 已采纳
- **上下文**: API 路由设计
- **决策**: 使用 URL 路径版本 + 模块命名
- **理由**: 清晰的路由组织，支持多版本
- **格式**: `/api/v1/{module}/{controller}/{action}`
- **影响**: 所有 Controller 需统一配置 Route
